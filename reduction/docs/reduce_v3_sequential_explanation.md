# CUDA Reduction V3: Sequential Addressing 详解

## 代码概览

```cuda
__global__ void reduce_v3(float *g_idata, float *g_odata, unsigned int n) {
    extern __shared__ float sdata[];
    unsigned int tid = threadIdx.x;
    unsigned int i = blockIdx.x * blockDim.x + threadIdx.x;

    sdata[tid] = (i < n) ? g_idata[i] : 0.0f;
    __syncthreads();

    for (unsigned int s = blockDim.x / 2; s > 0; s >>= 1) {
        if (tid < s) {
            sdata[tid] += sdata[tid + s];
        }
        __syncthreads();
    }

    if (tid == 0) atomicAdd(g_odata, sdata[0]);
}
```

---

## 核心改进：解决 Bank Conflict

### V2 vs V3 关键区别

| 特性 | V2 (Strided) | V3 (Sequential) |
|------|--------------|-----------------|
| **寻址模式** | `index = 2*s*tid` | `tid < s` |
| **循环方向** | 从小到大 (`s *= 2`) | 从大到小 (`s >>= 1`) |
| **内存访问** | 跳跃式（引起 Bank Conflict） | 连续式（无冲突） |
| **Warp Divergence** | 无 | 无 |
| **Bank Conflict** | ❌ 存在 | ✅ 消除 |

---

## 代码逐段解析

### 1. 数据加载（与前版本相同）

```cuda
extern __shared__ float sdata[];
unsigned int tid = threadIdx.x;
unsigned int i = blockIdx.x * blockDim.x + threadIdx.x;

sdata[tid] = (i < n) ? g_idata[i] : 0.0f;
__syncthreads();
```

### 2. 核心改进：反向归约循环

```cuda
for (unsigned int s = blockDim.x / 2; s > 0; s >>= 1) {
    if (tid < s) {
        sdata[tid] += sdata[tid + s];
    }
    __syncthreads();
}
```

**核心变化**：
- **V2**: `for (s = 1; s < blockDim.x; s *= 2)` → 从小到大
- **V3**: `for (s = blockDim.x / 2; s > 0; s >>= 1)` → 从大到小
- **V2**: `if (index < blockDim.x)` 其中 `index = 2*s*tid`
- **V3**: `if (tid < s)` → 更简单的条件，连续的线程活跃

## 为什么消除了 Bank Conflict？

### 什么是 Bank Conflict？

GPU 共享内存被划分为 32 个 **Bank**（存储体）。当同一 Warp 内的**多个线程同时访问同一个 Bank** 时，就会产生冲突，导致访问被串行化。

### V2 的 Bank Conflict 问题

**V2 访问模式**：`sdata[index] += sdata[index + s]` 其中 `index = 2*s*tid`

```
假设 8 个线程，Bank 数 = 4（简化说明）

第 1 轮 (s=1):
┌─────────────────────────────────────────────────────────────┐
│ Bank 0 │ Bank 1 │ Bank 2 │ Bank 3 │ Bank 0 │ Bank 1 │ Bank 2 │ Bank 3 │
│   0    │   1    │   2    │   3    │   4    │   5    │   6    │   7    │
│   ↑         ↑         ↑         ↑                              │
│  tid0      tid1      tid2      tid3   都在读取 Bank 0！       │
│ (sdata[0]) (sdata[2]) (sdata[4]) (sdata[6])                    │
└─────────────────────────────────────────────────────────────┘

冲突: tid0 访问 Bank 0 (地址0)，tid2 也访问 Bank 0 (地址4)
      → 2-way Bank Conflict！
```

### V3 的 Sequential Addressing 解决方案

**V3 访问模式**：`sdata[tid] += sdata[tid + s]` 其中连续的 `tid` 访问连续的地址

```
第 1 轮 (s=4):
┌─────────────────────────────────────────────────────────────┐
│ Bank 0 │ Bank 1 │ Bank 2 │ Bank 3 │ Bank 0 │ Bank 1 │ Bank 2 │ Bank 3 │
│   0    │   1    │   2    │   3    │   4    │   5    │   6    │   7    │
│   ↑      ↑        ↑        ↑                                 │
│  tid0   tid1     tid2     tid3                               │
│ (读0)   (读1)    (读2)    (读3)   每个线程访问不同的 Bank！   │
└─────────────────────────────────────────────────────────────┘

同时:
┌─────────────────────────────────────────────────────────────┐
│ Bank 0 │ Bank 1 │ Bank 2 │ Bank 3 │ Bank 0 │ Bank 1 │ Bank 2 │ Bank 3 │
│   0    │   1    │   2    │   3    │   4    │   5    │   6    │   7    │
│                        ↑       ↑       ↑       ↑             │
│                       tid0    tid1   tid2   tid3             │
│                      (读4)   (读5)  (读6)  (读7)              │
│                   tid0读Bank0, tid1读Bank1, tid2读Bank2...   │
│                   没有冲突！                                   │
└─────────────────────────────────────────────────────────────┘
```

**关键**：V3 中 `tid` 和 `tid+s` 访问的是**不同的 Bank**（只要 `s` 不是 Bank 大小的倍数）。

### Bank 访问对比表

| 版本 | 读取操作 | Bank 访问模式 | 冲突程度 |
|------|---------|--------------|---------|
| **V2** | `sdata[2*s*tid]`, `sdata[2*s*tid+s]` | 跳跃式，多个线程访问同一 Bank | 2-way |
| **V3** | `sdata[tid]`, `sdata[tid+s]` | 连续式，每个线程访问不同 Bank | 无冲突 |

## 完整对比：V1 → V2 → V3

### 代码对比

```cuda
// ============== V1: Interleaved Addressing ==============
for (unsigned int s = 1; s < blockDim.x; s *= 2) {
    if (tid % (2 * s) == 0) {           // 间隔寻址
        sdata[tid] += sdata[tid + s];
    }
    __syncthreads();
}
// 问题: Warp Divergence 严重

// ============== V2: Strided Addressing ==============
for (unsigned int s = 1; s < blockDim.x; s *= 2) {
    int index = 2 * s * tid;
    if (index < blockDim.x) {           // 顺序线程活跃
        sdata[index] += sdata[index + s];
    }
    __syncthreads();
}
// 改进: 消除 Warp Divergence
// 问题: Bank Conflict

// ============== V3: Sequential Addressing ==============
for (unsigned int s = blockDim.x / 2; s > 0; s >>= 1) {
    if (tid < s) {                      // 顺序线程活跃
        sdata[tid] += sdata[tid + s];
    }
    __syncthreads();
}
// 改进: 同时消除 Warp Divergence 和 Bank Conflict
```

### 性能演进

| 版本 | 寻址模式 | Warp Divergence | Bank Conflict | 相对性能 |
|------|---------|----------------|---------------|---------|
| **V1** | Interleaved | ❌ 严重 | ✅ 无 | 基准 (1x) |
| **V2** | Strided | ✅ 消除 | ❌ 存在 | ~2-3x |
| **V3** | Sequential | ✅ 消除 | ✅ 消除 | ~3-5x |

## 算法正确性验证

### 数学等价性

无论采用哪种寻址方式，最终的数学结果都是相同的：

```
输入: [1, 2, 3, 4, 5, 6, 7, 8]

V1 计算顺序: ((1+2)+(3+4))+((5+6)+(7+8)) = 36
V2 计算顺序: ((1+2)+(3+4))+((5+6)+(7+8)) = 36  
V3 计算顺序: ((1+5)+(2+6))+((3+7)+(4+8)) = 36

结果相同！只是结合顺序不同
```

### 结合律与浮点精度

虽然整数加法满足结合律，但**浮点数加法不满足结合律**：

```
(1e20 + (-1e20)) + 1.0 = 0 + 1.0 = 1.0
1e20 + ((-1e20) + 1.0) = 1e20 + (-1e20) = 0

不同顺序可能产生略微不同的结果！
```

**实际影响**：
- 对于归约求和，不同版本的 V1/V2/V3 可能产生略微不同的结果
- 这是浮点精度的固有特性，不是 bug
- 如果需要严格确定性，可以使用 Kahan 求和等算法

## 总结

| 特性 | 说明 |
|------|------|
| **算法名称** | Sequential Addressing（顺序寻址） |
| **核心改进** | 反向归约循环：`s = blockDim.x/2` 递减到 `s = 1` |
| **关键优势** | 同时消除 Warp Divergence 和 Bank Conflict |
| **内存访问** | 连续线程访问连续地址，无 Bank 冲突 |
| **进一步优化** | V4+ 版本将解决线程闲置问题（展开循环） |

V3 是归约算法的重要里程碑，通过简单的循环方向改变，同时解决了 V1 的 Warp Divergence 和 V2 的 Bank Conflict 两个问题，达到了相当优化的性能水平。
