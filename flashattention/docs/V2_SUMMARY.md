# V2 共享内存 KV 分块 - 快速参考

## 核心改进

V2 相比 V1 的核心改进是引入 **共享内存（Shared Memory）分块**，将 K/V 矩阵的全局内存访问减少 **64倍**。

```
┌─────────────────────────────────────────────────────────────┐
│  V1: 64个线程各自独立从全局内存加载K/V                      │
│      → 每个K/V行被加载64次                                  │
│      → 总内存访问: 64 × N × d                               │
├─────────────────────────────────────────────────────────────┤
│  V2: 64个线程协作加载K/V tile到共享内存                     │
│      → 每个K/V tile只加载1次                                │
│      → 所有线程复用共享内存中的数据                         │
│      → 总内存访问: 1 × N × d                                │
├─────────────────────────────────────────────────────────────┤
│  结果: 64倍减少全局内存访问！                                 │
└─────────────────────────────────────────────────────────────┘
```

## 关键代码段

### 1. 共享内存分配
```cuda
extern __shared__ float shared_mem[];
float *K_tile = shared_mem;                    // 起始位置
float *V_tile = shared_mem + V2_Bc * d;        // 偏移Bc×d
```

### 2. 协作加载
```cuda
int load_per_thread = (V2_Bc * d + V2_Br - 1) / V2_Br;  // 64

for (int i = 0; i < load_per_thread; i++) {
    int idx = tid * load_per_thread + i;
    if (idx < V2_Bc * d) {
        int row = idx / d;        // tile内的行
        int col = idx % d;        // tile内的列
        int global_row = kv_start + row;

        if (global_row < N) {
            K_tile[row * d + col] = K[global_row * d + col];
        } else {
            K_tile[row * d + col] = 0.0f;  // 越界填充0
        }
    }
}
```

### 3. 同步与计算
```cuda
__syncthreads();  // 等待所有线程完成加载

// 所有线程从共享内存读取并计算
for (int b = 0; b < V2_Bc; b++) {
    float qk = 0.0f;
    for (int i = 0; i < d; i++) {
        qk += q_vec[i] * K_tile[b * d + i];  // 共享内存访问！
    }
    // ... Online Softmax 和输出累加
}

__syncthreads();  // 等待所有线程用完当前tile
```

## 执行流程

```
对于每个Block:
  1. 每个线程加载自己的Q行到寄存器 (全局内存)

  对于每个KV tile (ceil(N/64)个tiles):
    2. 所有64个线程协作加载K tile到共享内存
    3. 所有64个线程协作加载V tile到共享内存
    4. __syncthreads() - 确保加载完成

    5. 每个线程处理这个tile的64个KV对:
       - 从共享内存读取K行
       - 计算qk
       - Online Softmax更新
       - 从共享内存读取V行
       - 累加到o_acc

    6. __syncthreads() - 确保用完tile

  7. 每个线程写回最终结果到O (全局内存)
```

## 常见问题

### Q: 为什么不对Q分块？
A: 每个线程只需要自己的一行Q，不需要与其他线程共享。Q从全局内存加载一次后存储在寄存器中，已经足够高效。

### Q: __syncthreads 可以省略吗？
A: 不可以！必须确保所有线程完成加载后才能使用共享内存，否则会有数据竞争。

### Q: 共享内存越大越好吗？
A: 不是！共享内存越大，每SM能同时运行的block数越少，occupancy降低。需要平衡。
