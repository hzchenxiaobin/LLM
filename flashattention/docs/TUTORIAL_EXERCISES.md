## 参考答案要点

> 以下为要点而非完整答卷，便于自查。

**§1**  
- 公式 $O=\mathrm{softmax}(QK^T/\sqrt{d})V$；$S,P$ 为 $N\times N$，$O$ 为 $N\times d$。  
- 近似方法改变注意力分布或引入不规则访存；FA 为精确、IO 感知实现。  
- Eager 多步内核使 $S,P$ 反复落盘与再读，HBM 流量大。

**§2**  
- HBM：大容量、带宽相对较低；SRAM：每 SM 小、带宽高；寄存器：极小、最快。  
- Memory-bound：算力等待数据，强度低。  
- 融合目标：尽量少次读 $Q,K,V$，只写 $O$，不在 HBM 存 $N\times N$ 中间矩阵。

**§3**  
- 行上维护 $m$（行最大）、$l$（缩放后指数和），$O(N)$。  
- $l^{\text{new}} = l^{(1)} e^{m^{(1)}-m^{\text{new}}} + \sum e^{x^{(2)}-m^{\text{new}}}$；旧部分需按新全局最大重标度。  
- 反向：不存 $P$，按块重算 $S$ 再得 $P$；多用算力、少 HBM 读大阵。

**§4**  
- V1：外 $K,V$ 内 $Q$；V2：外 $Q$ 内 $K,V$。  
- V2：Warp 私有寄存器、少 Warp 间同步。  
- 序列并行：batch 小、序列长时仍填满 SM。  
- 内层尽量纯 GEMM，喂饱 Tensor Core。

**§5**  
- TMA：异步搬数据；WGMMA：异步大块 MMA。  
- Softmax 与下一 GEMM 重叠；块量化缓解 FP8 离群值。

**§6**  
- TMEM：张量核心友好片上存储；2-CTA 减少原子归约。  
- SFU $\exp$ 吞吐相对 TC 过低 → 多项式/FMA 模拟。  
- CuTeDSL：缩短编译时间、Python 侧 DSL。

**§7**  
- $Q$ 为 $1\times d$；GEMV、SM 利用率低。  
- 沿 KV 序列切分；局部 $m,l$（及未归一化贡献）再全局归约。  
- MBA：运行时多块调度，应对超长上下文显存与延迟。

**§8**  
- 稀疏/哈希等破坏合并访存；FA 分块稠密、合并访存友好；数学上仍为精确 softmax。

**§9**  
- 例：128K、1M token 等；MInference：先估活跃块再 FA 精确算。

**§10**  
- `torch.nn.functional.scaled_dot_product_attention`。  
- 浮点非结合律、分块顺序与重缩放不同 → 微小差异；通常可接受。

**§11**  
- V1：$O(N^2)$ HBM 中间阵；V2：Warp/序列并行与循环；V3：Hopper 异步；V4：Blackwell TMEM/UMMA/SFU 瓶颈。  
- 术语链示例：HBM 带宽瓶颈 → 分块只在 SRAM 完成局部注意力 → $O(N)$ 统计量替代物化 $P$。
