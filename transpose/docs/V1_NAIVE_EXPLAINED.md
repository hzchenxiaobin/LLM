# V1: 朴素转置实现详解

## 概述

这是 CUDA 矩阵转置的最基础实现，采用**最直接**的方式：每个线程处理一个元素。

---

## 完整代码

```cpp
#include "transpose_common.h"

// v1: 朴素实现
// 每个线程处理一个元素
// 读取A是合并访问，写入B是非合并访问
__global__ void transpose_naive_kernel(const float *A, float *B, int N) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;  // 列索引
    int y = blockIdx.y * blockDim.y + threadIdx.y;  // 行索引

    if (x < N && y < N) {
        // A: 连续读取 (y * N + x) - 合并访问
        // B: 非连续写入 (x * N + y) - 非合并访问
        B[x * N + y] = A[y * N + x];
    }
}

void transpose_naive(const float *d_A, float *d_B, int N, cudaStream_t stream) {
    dim3 block(32, 32);
    dim3 grid((N + block.x - 1) / block.x, (N + block.y - 1) / block.y);
    transpose_naive_kernel<<<grid, block, 0, stream>>>(d_A, d_B, N);
}
```

---

## 核心逻辑解析

### 1. 线程索引计算

```cpp
int x = blockIdx.x * blockDim.x + threadIdx.x;  // 列索引 (0 ~ N-1)
int y = blockIdx.y * blockDim.y + threadIdx.y;  // 行索引 (0 ~ N-1)
```

每个线程通过二维 CUDA 索引计算自己在矩阵中的位置：
- `x`：矩阵的列索引（横向位置）
- `y`：矩阵的行索引（纵向位置）

### 2. 数据转置操作

```cpp
B[x * N + y] = A[y * N + x];
```

将 A 中位置 `(y, x)` 的元素复制到 B 的位置 `(x, y)`，完成转置。

## 关键问题：非合并内存访问

### 读取矩阵 A（✅ 合并访问 Coalesced）

```
Warp 内的 32 个线程（同一行，threadIdx.x = 0~31）同时读取：

线程 0: 读取 A[y*N + 0]  → 地址: y*N
线程 1: 读取 A[y*N + 1]  → 地址: y*N + 1
线程 2: 读取 A[y*N + 2]  → 地址: y*N + 2
 ...
线程 31: 读取 A[y*N + 31] → 地址: y*N + 31

内存访问模式：
地址: y*N  y*N+1  y*N+2  ...  y*N+31
      │     │      │           │
      ▼     ▼      ▼           ▼
    ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐
    │ 0 │ 1 │ 2 │ 3 │...│ 29│ 30│ 31│   │  ← 连续内存区域
    └───┴───┴───┴───┴───┴───┴───┴───┴───┘
    
✅ 一次内存事务即可完成 32 个 float 的读取！
   总线利用率: 100%
```

### 写入矩阵 B（❌ 非合并访问 Strided）

```
Warp 内的 32 个线程（同一行）同时写入 B：

线程 0: 写入 B[x*N + y] 其中 x=0  → 地址: 0*N + y = y
线程 1: 写入 B[x*N + y] 其中 x=1  → 地址: 1*N + y = N + y
线程 2: 写入 B[x*N + y] 其中 x=2  → 地址: 2*N + y = 2N + y
 ...
线程 31: 写入 B[x*N + y] 其中 x=31 → 地址: 31*N + y

内存访问模式（假设 N=1024, y=5）：
地址:    5     1029    2053    3077    ...   31741
         │       │       │       │            │
         ▼       ▼       ▼       ▼            ▼
    ┌───────┐ ┌───────┐ ┌───────┐          ┌───────┐
    │  thr0 │ │  thr1 │ │  thr2 │   ...    │ thr31 │
    └───┬───┘ └───┬───┘ └───┬───┘          └───┬───┘
        │         │         │                  │
        ▼         ▼         ▼                  ▼
    地址:5    地址:1029  地址:2053          地址:31741
    
跨度 = N × sizeof(float) = 4096 字节

❌ 每次写入间隔 4096 字节，需要 32 次独立的内存事务！
   总线利用率: ~12.5% (128B cache line / 4096B stride)
```

## 启动配置详解

```cpp
void transpose_naive(const float *d_A, float *d_B, int N, cudaStream_t stream) {
    dim3 block(32, 32);           // 每个 Block: 32×32 = 1024 线程
    dim3 grid((N + block.x - 1) / block.x,   // X 方向 Block 数
              (N + block.y - 1) / block.y);  // Y 方向 Block 数
    transpose_naive_kernel<<<grid, block, 0, stream>>>(d_A, d_B, N);
}
```

### Grid 维度计算

```
当 N = 4096 时:

grid.x = (4096 + 32 - 1) / 32 = 4127 / 32 = 129
grid.y = (4096 + 32 - 1) / 32 = 129

Grid: 129 × 129 = 16641 个 Block
每个 Block: 1024 线程
总线程数: 16641 × 1024 = 17,040,384 线程

矩阵元素数: 4096 × 4096 = 16,777,216

(多出的线程由边界检查 if (x < N && y < N) 过滤)
```

## 性能特征

| 指标 | 数值/描述 |
|------|-----------|
| **计算复杂度** | O(N²) |
| **内存访问** | 读取: N² 次, 写入: N² 次 |
| **读取模式** | ✅ 合并访问 (高带宽利用率) |
| **写入模式** | ❌ 非合并访问 (低带宽利用率) |
| **理论带宽占比** | 峰值带宽的 **10%~20%** |
| **主要瓶颈** | 写入端的非合并内存访问 |

## 总结

### 优点
1. **代码简单直观**，容易理解和实现
2. **读取端是合并访问**，充分利用读取带宽
3. **正确性保证**，能正确处理任意大小的矩阵

### 缺点
1. **写入端非合并访问**，严重浪费写入带宽
2. **仅能达到峰值带宽的 10%~20%**
3. **未利用共享内存**进行数据中转和重排

### 后续优化方向
下一版本 (`v2_shared_memory.cu`) 将引入 **Shared Memory** 进行分块转置，通过以下步骤解决非合并写入问题：
1. 合并读取 A 到 Shared Memory
2. 在 Shared Memory 中完成转置（无全局内存压力）
3. 合并写入 B

这将显著提升写入带宽利用率。
