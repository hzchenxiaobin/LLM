# Blelloch Scan 算法详解 (v2_blelloch.cu)

## 概述

Blelloch Scan 是一种**工作高效(Work-Efficient)**的并行前缀和算法，由 Guy Blelloch 在 1989 年提出。与 Hillis-Steele 算法相比，它将总工作量从 O(N log N) 降低到 O(N)，达到了与串行算法相同的渐进复杂度。

---

## 核心思想

算法分为两个主要阶段：

1. **Up-Sweep (归约阶段)**：自底向上构建部分和树
2. **Down-Sweep (分发阶段)**：自顶向下计算前缀和

```
┌─────────────────────────────────────────────────────────────────┐
│                    Blelloch Scan 流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  输入: [1, 2, 0, 1, 2, 0, 1, 2]                                 │
│                          │                                      │
│                          ▼                                      │
│  ┌──────────────────────────────────────┐                      │
│  │     Up-Sweep (归约阶段)              │                      │
│  │  构建树结构，计算部分和              │                      │
│  └──────────────────────────────────────┘                      │
│                          │                                      │
│                          ▼                                      │
│  部分和树 (根=总和9)                                           │
│                          │                                      │
│                          ▼                                      │
│  ┌──────────────────────────────────────┐                      │
│  │     Down-Sweep (分发阶段)            │                      │
│  │  根节点归零，向下分发前缀和          │                      │
│  └──────────────────────────────────────┘                      │
│                          │                                      │
│                          ▼                                      │
│  输出: [0, 1, 3, 3, 4, 6, 6, 7]  (排他扫描)                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 阶段一：Up-Sweep (归约树)

### 可视化

```

                    │    9    │  ← 根节点 (总和)
                    │ (root)  │
                    └────┬────┘
           ┌─────────────┴─────────────┐
           ▼                           ▼
      ┌─────────┐                 ┌─────────┐
      │    4    │                 │    5    │
      │ (1+3)   │                 │ (2+3)   │
      └────┬────┘                 └────┬────┘
     ┌─────┴─────┐               ┌─────┴─────┐
     ▼           ▼               ▼           ▼
┌─────────┐ ┌─────────┐    ┌─────────┐ ┌─────────┐
│    3    │ │    1    │    │    2    │ │    3    │
│  (1+2)  │ │  (0+1)  │    │  (2+0)  │ │  (1+2)  │
└────┬────┘ └────┬────┘    └────┬────┘ └────┬────┘
   ┌─┴─┐       ┌─┴─┐          ┌─┴─┐       ┌─┴─┐
   ▼   ▼       ▼   ▼          ▼   ▼       ▼   ▼
┌───┐┌───┐  ┌───┐┌───┐     ┌───┐┌───┐  ┌───┐┌───┐
│ 1 ││ 2 │  │ 0 ││ 1 │     │ 2 ││ 0 │  │ 1 ││ 2 │ ← 输入
└───┘└───┘  └───┘└───┘     └───┘└───┘  └───┘└───┘

图例:
┌───┐ 绿色/蓝色圆圈 = 部分和节点
│   │ 叶子节点 = 输入值
└───┘
```

### 代码实现

```cuda
// ========== 1. Up-Sweep 阶段 (归约) ==========
// 从叶子节点向上归约，构建部分和树
for (int d = n >> 1; d > 0; d >>= 1) {
    __syncthreads();
    if (thid < d) {
        int ai_local = offset * (2 * thid + 1) - 1;
        int bi_local = offset * (2 * thid + 2) - 1;
        temp[bi_local] += temp[ai_local];
    }
    offset *= 2;
}
```

### 关键要点

| 属性 | 说明 |
|------|------|
| **迭代次数** | log₂N 次 |
| **每次活跃线程** | n/2, n/4, n/8, ..., 1 |
| **操作** | 右子节点 += 左子节点 |
| **结果** | 根节点存储总和 |

### 索引计算

```
对于第 d 次迭代 (d = n/2, n/4, n/8, ...):

  ai = offset × (2×thid + 1) - 1  ← 左索引
  bi = offset × (2×thid + 2) - 1  ← 右索引

示例 (n=8, offset=1, d=4):
  thid=0: ai=0, bi=1  → temp[1] += temp[0]
  thid=1: ai=2, bi=3  → temp[3] += temp[2]
  thid=2: ai=4, bi=5  → temp[5] += temp[4]
  thid=3: ai=6, bi=7  → temp[7] += temp[6]
```

## 阶段二：Down-Sweep (分发树)

### 关键步骤：根节点归零

```
排他扫描的关键！

原始根节点:          清零后:
┌─────────┐         ┌─────────┐
│    9    │   →→→   │    0    │  ← temp[n-1] = 0
│ (总和)  │         │ (归零)  │
└─────────┘         └─────────┘
```

### 可视化 - 交换与累加

```
Down-Sweep 核心操作：交换并累加

Step 0: 根节点清零
                    ┌─────────┐
                    │    0    │  ← 根节点设为0 (排他扫描)
                    └────┬────┘
           ┌─────────────┴─────────────┐
           │                           │
      ┌────┴────┐                 ┌────┴────┐
      │    4    │                 │    5    │
      │(原值)   │                 │(原值)   │
      └─────────┘                 └─────────┘

Step 1: 第一层分发 (offset=4)
                    ┌─────────┐
                    │    0    │  ← 保存的值 t=4
                    └────┬────┘
           ┌─────────────┴─────────────┐
           ▼                           ▼
      ┌─────────┐                 ┌─────────┐
      │    0    │ ←──────────────┤    4    │ 交换: ai=3, bi=7
      │(新ai)   │                │(新bi)   │ temp[3]=0, temp[7]=0+4=4
      └─────────┘                 └─────────┘

Step 2: 第二层分发 (offset=2)
      ┌─────────┐                 ┌─────────┐
      │    0    │                 │    4    │
      └────┬────┘                 └────┬────┘
     ┌─────┴─────┐               ┌─────┴─────┐
     ▼           ▼               ▼           ▼
┌─────────┐ ┌─────────┐    ┌─────────┐ ┌─────────┐
│    0    │ │    3    │    │    4    │ │    6    │
│(新ai)   │ │(新ai)   │    │(新bi)   │ │(新bi)   │
└─────────┘ └─────────┘    └─────────┘ └─────────┘
交换:          交换:             累加:        累加:
t=3↔0       t=2↔4            0+3=3       4+2=6

Step 3: 最终层 (offset=1, 叶节点 = 排他扫描结果)
   ┌─┐       ┌─┐          ┌─┐       ┌─┐
   ▼ ▼       ▼ ▼          ▼ ▼       ▼ ▼
┌───┐┌───┐┌───┐┌───┐  ┌───┐┌───┐┌───┐┌───┐
│ 0 ││ 1 ││ 3 ││ 3 │  │ 4 ││ 6 ││ 6 ││ 7 │ ← 排他扫描结果!
└───┘└───┘└───┘└───┘  └───┘└───┘└───┘└───┘
   0   1   3   3        4   6   6    7
   ↓   ↓   ↓   ↓        ↓   ↓   ↓    ↓
  1前 2前 0前 1前      2前 0前 1前  2前
  和=0 和=1 和=3 和=3   和=4 和=6 和=6 和=7

Step 4: 转换为包含扫描 (可选)
排他 + 原数组 = [0+1, 1+2, 3+0, 3+1, 4+2, 6+0, 6+1, 7+2]
              = [  1,   3,   3,   4,   6,   6,   7,   9]
```

### 代码实现

```cuda
// 将根节点设为 0 (这是排他型扫描的关键)
if (thid == 0) {
    temp[n - 1] = 0;
}

// ========== 2. Down-Sweep 阶段 (分发) ==========
// 自顶向下计算前缀和
for (int d = 1; d < n; d *= 2) {
    offset >>= 1;
    __syncthreads();
    if (thid < d) {
        int ai_local = offset * (2 * thid + 1) - 1;
        int bi_local = offset * (2 * thid + 2) - 1;

        // 交换并累加：
        // - ai 位置获得原来的 bi 值（当前前缀和）
        // - bi 位置累加上 ai 的值（传播前缀和）
        T t = temp[ai_local];
        temp[ai_local] = temp[bi_local];
        temp[bi_local] += t;
    }
}
```

### 核心操作解析

```
关键三行代码的魔力:

    T t = temp[ai];           // 1. 保存左节点的值
    temp[ai] = temp[bi];      // 2. 左节点获得右节点的值(前缀和)
    temp[bi] += t;             // 3. 右节点累加左节点的值

示意图:
    操作前:              操作后:
    ┌─────┐┌─────┐      ┌─────┐┌─────┐
    │ ai  ││ bi  │  →→  │ bi  ││bi+ai│
    │ =A  ││ =B  │      │ 旧 ││ 新  │
    └─────┘└─────┘      └─────┘└─────┘
                              ↑
                         左节点获得
                         原来的右值
```

## 复杂度分析

### 工作量 (Work)

```
Up-Sweep 工作量:
  n/2 + n/4 + n/8 + ... + 1 = n - 1 次加法

Down-Sweep 工作量:
  1 + 2 + 4 + ... + n/2 = n - 1 次操作(交换+加法)

总工作量: O(n)
```

### 跨度 (Span)

```
两个阶段的迭代次数都是 log₂n

总跨度: O(log n)
```

### 效率对比

| 算法 | 工作量 | 跨度 | 工作高效? |
|------|--------|------|-----------|
| **串行扫描** | O(n) | O(n) | ✅ 基准 |
| **Hillis-Steele** | O(n log n) | O(log n) | ❌ 工作量超标 |
| **Blelloch Scan** | **O(n)** | O(log n) | ✅ **最优** |

## 排他扫描 vs 包含扫描

### 排他扫描 (Exclusive Scan)

```
输入:  [1, 2, 0, 1, 2, 0, 1, 2]
输出:  [0, 1, 3, 3, 4, 6, 6, 7]
       ↑
     第i个输出 = 前i个输入的和
     output[i] = sum(input[0..i-1])

验证:
- output[0] = 0                    (1之前的和)
- output[1] = 1                    (2之前的和=1)
- output[2] = 1+2 = 3              (0之前的和=3)
- output[3] = 1+2+0 = 3            (1之前的和=3)
- output[4] = 1+2+0+1 = 4          (2之前的和=4)
- output[5] = 1+2+0+1+2 = 6        (0之前的和=6)
- output[6] = 1+2+0+1+2+0 = 6      (1之前的和=6)
- output[7] = 1+2+0+1+2+0+1 = 7    (2之前的和=7)
```

### 转换为包含扫描 (Inclusive Scan)

```
如果需要包含扫描，只需最后加一次:

排他结果: [0, 1, 3, 3, 4, 6, 6, 7]
原始输入: [1, 2, 0, 1, 2, 0, 1, 2]
         ↓ 逐元素相加
包含结果: [1, 3, 3, 4, 6, 6, 7, 9]
           ↑              ↑
         0+1=1          6+1=7
```

## 关键代码解析

### 1. 共享内存使用

```cuda
extern __shared__ T temp[];  // 动态分配的共享内存

// 加载数据到共享内存 (每个线程2个元素)
int ai = 2 * thid;
int bi = 2 * thid + 1;
temp[ai] = (ai < n) ? g_idata[ai] : 0;
temp[bi] = (bi < n) ? g_idata[bi] : 0;
```

### 2. 线程同步

```cuda
__syncthreads();  // 每次迭代后必须同步！
                  // 确保所有线程完成当前层
                  // 才能进行下一层计算
```

### 3. 活跃线程控制

```cuda
if (thid < d) {   // 只有前d个线程活跃
    // ...       // d 每次减半(Up-Sweep)或倍增(Down-Sweep)
}
```

## 局限性

1. **必须是2的幂次**: 算法假设 n 是 2的幂次
2. **单block限制**: 当前实现使用单个block，最大处理1024个线程(2048个元素)
3. **需要全局扫描**: 大规模数据需要分块+全局扫描

---

## 总结

Blelloch Scan 通过巧妙的**两阶段树形归约**，实现了：
- ✅ **工作高效**: O(N) 工作量，与串行算法相同
- ✅ **并行友好**: O(log N) 跨度，适合GPU并行
- ✅ **空间局部性**: 使用共享内存，减少全局内存访问

**核心洞察**: 通过先构建部分和树(Up-Sweep)，再反向分发(Down-Sweep)，避免了重复计算，达到了工作量的理论下界。
