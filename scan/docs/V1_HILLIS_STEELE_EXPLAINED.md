# V1: Hillis-Steele 朴素并行扫描 - 详细解析

## 概述

Hillis-Steele 算法是一种**直观的并行扫描算法**，通过逐步倍增的累加方式计算前缀和。虽然工作量不是最优（O(N log N)），但实现简单，是理解 CUDA 并行扫描的良好起点。

---

## 算法原理

### 核心思想

对于长度为 N 的数组，算法执行 **log₂N 轮迭代**。在第 d 轮迭代中，每个线程将其当前元素与距离它 **2^(d-1)** 的元素相加。

**以输入 `[3, 1, 7, 0, 4, 1, 6, 3]` 为例：**

```
初始状态:  [3, 1, 7, 0, 4, 1, 6, 3]

第 1 轮 (offset=1):  与相邻元素相加
  [3, 1, 7, 0, 4, 1, 6, 3]
   ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓
  [3, 4, 8, 7, 4, 5, 7, 9]
   3+0  1+3  7+1  0+7  4+0  1+4  6+1  3+6

第 2 轮 (offset=2):  与距离 2 的元素相加
  [3, 4, 8, 7, 4, 5, 7, 9]
   ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓
  [3, 4, 11, 11, 12, 12, 11, 16]
   3+0  4+0  8+3  7+4  4+8  5+7  7+4  9+7

第 3 轮 (offset=4):  与距离 4 的元素相加
  [3, 4, 11, 11, 12, 12, 11, 16]
   ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓
  [3, 4, 11, 11, 15, 16, 22, 27]
   3+0  4+0  11+0  11+0  12+3  12+4  11+11  16+11

包含型结果: [3, 4, 11, 11, 15, 16, 22, 27]
排他型结果: [0, 3, 4, 11, 11, 15, 16, 22]  (右移一位，首位补 0)
```

## 双缓冲技术详解

### 为什么需要双缓冲？

在并行计算中，多个线程同时读写共享内存会导致**数据竞争（Data Race）**。Hillis-Steele 算法使用**双缓冲（Double Buffering）**技术来解决这个问题。

```
┌─────────────────────────────────────────────────────────┐
│                    共享内存布局                          │
├─────────────────────────────────────────────────────────┤
│  Buffer 0 (pout=0)    │    Buffer 1 (pin=1)            │
│  ┌─────┬─────┬─────┬───┴───┬─────┬─────┬─────┬─────┐   │
│  │  3  │  1  │  7  │   0   │  4  │  1  │  6  │  3  │   │
│  └─────┴─────┴─────┴───────┴─────┴─────┴─────┴─────┘   │
│     ↑                                    ↑              │
│   读取                                  写入            │
│  (上一轮结果)                           (当前轮结果)     │
└─────────────────────────────────────────────────────────┘

每轮迭代后交换 pout 和 pin，读写角色互换
```

### 双缓冲索引计算

```cpp
int pout = 0, pin = 1;  // 初始：pout=写入, pin=读取

for (int offset = 1; offset < n; offset *= 2) {
    pout = 1 - pout;    // 交换: pout 变为 1
    pin = 1 - pout;     // pin 变为 0
    // 现在: pout=1(写入), pin=0(读取)
    
    // 访问索引: temp[pout * n + thid] = ...
    //           temp[pin * n + thid - offset] = ...
}
```

## 代码逐行解析

### 1. 包含型扫描内核

```cpp
template <typename T>
__global__ void hillis_steele_kernel(T* g_odata, const T* g_idata, int n) {
    // 分配动态共享内存，大小为 2 * n * sizeof(T)
    // 双缓冲需要两倍空间
    extern __shared__ T temp[];
    
    int thid = threadIdx.x;     // 当前线程在线程块中的索引
    int pout = 0, pin = 1;      // pout=写入缓冲区索引, pin=读取缓冲区索引

    // 将数据从 Global Memory 读入 Shared Memory
    // 使用条件判断处理边界情况（当 n 不是线程数的整数倍时）
    temp[pout * n + thid] = (thid < n) ? g_idata[thid] : 0;
    __syncthreads();            // 确保所有线程都完成加载

    // 迭代进行扫描，offset 每次翻倍 (1, 2, 4, 8...)
    // 需要 log₂N 轮迭代
    for (int offset = 1; offset < n; offset *= 2) {
        // 交换双缓冲索引
        // pout 和 pin 在 0 和 1 之间交替切换
        pout = 1 - pout;
        pin = 1 - pout;

        // 如果当前线程索引 >= offset，则可以进行累加
        // 否则保持原值不变
        if (thid >= offset) {
            // 当前值 = 原位置值 + 前面 offset 位置的值
            temp[pout * n + thid] = temp[pin * n + thid] + temp[pin * n + thid - offset];
        } else {
            // 线程索引 < offset 时，无法找到前面 offset 位置的元素
            // 直接复制原值
            temp[pout * n + thid] = temp[pin * n + thid];
        }
        __syncthreads();        // 确保所有线程完成本轮计算
    }

    // 将最终结果写回 Global Memory
    // 注意：写入的是 pout 指向的缓冲区（最后写入的那个）
    if (thid < n) {
        g_odata[thid] = temp[pout * n + thid];
    }
}
```

### 2. 排他型扫描内核

```cpp
template <typename T>
__global__ void hillis_steele_exclusive_kernel(T* g_odata, const T* g_idata, int n) {
    extern __shared__ T temp[];
    int thid = threadIdx.x;
    int pout = 0, pin = 1;

    // 加载数据到共享内存（包含型扫描）
    temp[pout * n + thid] = (thid < n) ? g_idata[thid] : 0;
    __syncthreads();

    // Hillis-Steele 包含型扫描（与上面相同）
    for (int offset = 1; offset < n; offset *= 2) {
        pout = 1 - pout;
        pin = 1 - pout;

        if (thid >= offset) {
            temp[pout * n + thid] = temp[pin * n + thid] + temp[pin * n + thid - offset];
        } else {
            temp[pout * n + thid] = temp[pin * n + thid];
        }
        __syncthreads();
    }

    // 转换为排他型扫描：右移，首元素置 0
    if (thid < n) {
        T inclusive_result = temp[pout * n + thid];      // 包含型结果
        // 排他型结果 = 前一个位置的包含型结果，thid=0 时为 0
        T exclusive_result = (thid == 0) ? 0 : temp[pout * n + thid - 1];
        g_odata[thid] = exclusive_result;
    }
}
```

### 3. 主机端包装函数

```cpp
void hillis_steele_scan(const float* d_input, float* d_output, int n) {
    // 限制 block 大小不超过 1024 (CUDA 单 Block 最大线程数)
    // 取 n 和 1024 的最小值
    int threads = min(n, 1024);

    // 计算需要的共享内存大小：2 * n * sizeof(float)
    // 2 是因为双缓冲需要两个缓冲区
    int smem_size = 2 * n * sizeof(float);

    // 启动内核：1 个 Block，threads 个线程，smem_size 共享内存
    // 使用 hillis_steele_exclusive_kernel 计算排他型扫描
    hillis_steele_exclusive_kernel<<<1, threads, smem_size>>>(d_output, d_input, n);

    cudaDeviceSynchronize();    // 等待 GPU 完成计算
}
```

## 复杂度分析

### 工作量 (Work)

```
总工作量 = N × log₂N

对于 N = 1024:
- 第 1 轮: 1024 次操作 (每个线程读取 1 次，部分写入)
- 第 2 轮: 1024 次操作
- 第 3 轮: 1024 次操作
- ...共 log₂(1024) = 10 轮

总操作 ≈ 1024 × 10 = 10240 次加法
```

对比最优算法（如 Blelloch）的 O(N) = 1024，Hillis-Steele 多做了约 10 倍工作。

### 跨度 (Span/Critical Path)

```
跨度 = log₂N

最长依赖链是 log₂N，因为每轮迭代依赖于上一轮的结果
```

### 空间复杂度

```
共享内存使用 = 2 × N × sizeof(T)

双缓冲需要两倍的存储空间
```

## 优缺点总结

| 特性 | Hillis-Steele |
|------|---------------|
| **工作量** | O(N log N) - 非最优 |
| **跨度** | O(log N) |
| **共享内存** | 2N - 较高 |
| **实现复杂度** | 简单 |
| **同步次数** | log N 次 __syncthreads |
| **Bank Conflict** | 可能有（取决于 N）|

### 优点
1. **实现简单**：逻辑直观，代码易懂
2. **并行度高**：每轮所有线程都可以工作
3. **适合教学**：理解扫描算法的良好起点

### 缺点
1. **工作效率低**：O(N log N) 而非 O(N)
2. **内存占用高**：需要 2N 的共享内存
3. **同步开销**：需要 log N 次全局同步

---

## 使用建议

1. **学习目的**：Hillis-Steele 是理解并行扫描的最佳起点
2. **小规模数据**：N < 256 时，效率差异不明显
3. **生产环境**：建议使用 Blelloch 算法或 NVIDIA CUB 库
4. **内存受限**：如果共享内存紧张，考虑 Blelloch (只需 N 空间)
