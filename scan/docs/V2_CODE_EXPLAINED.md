# V2 Blelloch Scan 代码详解

## 概述

`v2_blelloch.cu` 实现了工作高效的前缀和算法 (Work-Efficient Scan)，相比 V1 Hillis-Steele 算法的 O(N log N) 工作量，Blelloch 算法将工作量优化到 **O(N)**。

---

## 算法核心思想

Blelloch 算法分为两个阶段：

1. **Up-Sweep (归约阶段)**：自底向上构建部分和树
2. **Down-Sweep (分发阶段)**：自顶向下计算前缀和

通过二叉树结构组织计算，每个内部节点存储子树的累加和。

---

## Up-Sweep 阶段详解

### 计算流程图

```
                    Up-Sweep: 自底向上归约


输入:        3       1       7       0       4       1
             |       |       |       |       |       |
             v       v       v       v       v       v

                     +------------------+
                     |  temp[7] = 16    |  <- 根节点 (总和)
                     |  [0..5]累加      |
                     +--------+---------+
                              |
           +------------------+------------------+
           |                                     |
 +---------+---------+                 +---------+---------+
 | temp[3] = 11      |  <- Level 2     | temp[7] = 5       |
 | [0..3]累加        |                 | [4..5]累加+填充   |
 +---------+---------+                 +---------+---------+
           |                                     |
 +---------+---------+                 +---------+
 |                   |                 |
+---+-----+   +-----+-----+   +-----+-----+
|temp[1]=4|   |temp[3]=7  |   |temp[5]=5  |
|[0..1]   |<-L1| [2..3]   |<-L1| [4..5]   |<-L1
+---+-----+   +-----+-----+   +-----+-----+
    |               |               |
    |       +-------+               |
    |       |                       |
+---+-----+-----+ +-----+-----+ +-----+-----+ +-----+-----+ +-----+-----+ +-----+-----+
|             | |         | |         | |         | |         | |         |
| temp[0]=3   | |temp[1]=1| |temp[2]=7| |temp[3]=0| |temp[4]=4| |temp[5]=1|
|  a[0]       | |  a[1]   | |  a[2]   | |  a[3]   | |  a[4]   | |  a[5]   |
|  叶子       | |  叶子   | |  叶子   | |  叶子   | |  叶子   | |  叶子   |
+-------------+ +---------+ +---------+ +---------+ +---------+ +---------+

                                                             +---------+ +---------+
                                                             |temp[6]=0| |temp[7]=0|
                                                             | (填充)  | |  (根)   |
                                                             +---------+ +---------+
```

### 代码对应

```cpp
// 每个线程加载两个元素到 Shared Memory
int ai = 2 * thid;           // 偶数索引
int bi = 2 * thid + 1;       // 奇数索引
temp[ai] = (ai < n) ? g_idata[ai] : 0;
temp[bi] = (bi < n) ? g_idata[bi] : 0;
```

**关键概念**:
- 每个线程处理 2 个元素
- 线程数 = n/2 (n 必须是 2 的幂次)
- `ai` 和 `bi` 分别对应左右子节点

### Up-Sweep 循环

```cpp
for (int d = n >> 1; d > 0; d >>= 1) {
    __syncthreads();
    if (thid < d) {
        int ai_local = offset * (2 * thid + 1) - 1;
        int bi_local = offset * (2 * thid + 2) - 1;
        temp[bi_local] += temp[ai_local];  // 右节点 += 左节点
    }
    offset *= 2;
}
```

**索引计算详解**:

| 轮次 | d (线程数) | offset | ai_local 公式 | bi_local 公式 |
|------|-----------|--------|---------------|---------------|
| 1    | n/2       | 1      | 2*thid        | 2*thid+1      |
| 2    | n/4       | 2      | 4*thid+1      | 4*thid+3      |
| 3    | n/8       | 4      | 8*thid+3      | 8*thid+7      |

**计算过程 (N=6)**:
```
Round 1 (d=4, offset=1):
  thid=0: ai=0, bi=1  → temp[1] += temp[0]  (1+3=4)
  thid=1: ai=2, bi=3  → temp[3] += temp[2]  (0+7=7)
  thid=2: ai=4, bi=5  → temp[5] += temp[4]  (1+4=5)
  thid=3: ai=6, bi=7  → temp[7] += temp[6]  (0+0=0)

Round 2 (d=2, offset=2):
  thid=0: ai=1, bi=3  → temp[3] += temp[1]  (7+4=11)
  thid=1: ai=5, bi=7  → temp[7] += temp[5]  (0+5=5)

Round 3 (d=1, offset=4):
  thid=0: ai=3, bi=7  → temp[7] += temp[3]  (5+11=16) ← 总和
```

## 根节点清零

```cpp
if (thid == 0) {
    temp[n - 1] = 0;  // temp[7] = 0
}
```

**为什么清零？**
- 排他型前缀和 (Exclusive Scan) 的定义：结果[i] = 输入[0] 到 输入[i-1] 的和
- 根节点代表"所有元素的和"
- 对于第一个元素，前面没有元素，所以应该是 0
- 清零为 Down-Sweep 阶段做准备

## Down-Sweep 阶段详解

### 核心操作：交换并累加

```cpp
for (int d = 1; d < n; d *= 2) {
    offset >>= 1;
    __syncthreads();
    if (thid < d) {
        int ai_local = offset * (2 * thid + 1) - 1;
        int bi_local = offset * (2 * thid + 2) - 1;

        T t = temp[ai_local];           // 保存左子树的值
        temp[ai_local] = temp[bi_local]; // 左位置获得父节点的值
        temp[bi_local] += t;              // 右位置累加左子树的值
    }
}
```

### 三行代码的魔力

```
操作前:              操作后:
┌─────┐ ┌─────┐     ┌─────┐ ┌─────┐
│ ai  │ │ bi  │  →  │ ai  │ │ bi  │
│  4  │ │  7  │     │  7  │ │ 11  │
└─────┘ └─────┘     └─────┘ └─────┘

变化:
  t = 4
  temp[ai] = 7    ← ai 获得"父节点的和"(排他型前缀和)
  temp[bi] = 7+4  ← bi 累加"左兄弟的值"
```

### Down-Sweep 计算过程 (N=6)

```
初始状态 (temp[7]=0):
  [3, 4, 7, 11, 4, 5, 0, 0]
   0   1   2   3   4   5   6   7

Round 1 (d=1, offset=4):
  thid=0: ai=3, bi=7
    t = temp[3] = 11
    temp[3] = temp[7] = 0
    temp[7] = 0 + 11 = 11
  结果: [3, 4, 7, 0, 4, 5, 0, 11]

Round 2 (d=2, offset=2):
  thid=0: ai=1, bi=3
    t = temp[1] = 4
    temp[1] = temp[3] = 0
    temp[3] = 0 + 4 = 4
  thid=1: ai=5, bi=7
    t = temp[5] = 5
    temp[5] = temp[7] = 11
    temp[7] = 11 + 5 = 16
  结果: [3, 0, 7, 4, 4, 11, 0, 16]

Round 3 (d=4, offset=1):
  thid=0: ai=0, bi=1  → temp[0]=0,  temp[1]=0+3=3
  thid=1: ai=2, bi=3  → temp[2]=4,  temp[3]=4+7=11
  thid=2: ai=4, bi=5  → temp[4]=11, temp[5]=11+4=15
  thid=3: ai=6, bi=7  → temp[6]=16, temp[7]=16+0=16
  
最终结果 (排他型前缀和):
  [0, 3, 4, 11, 11, 15, 16, 0]
   ↑  ↑  ↑  ↑  ↑  ↑
   有效结果 (前6个)
```

## 完整数据流图示

```
输入:              [3,  1,  7,  0,  4,  1]
                   ↓   ↓   ↓   ↓   ↓   ↓
Shared Memory:    [3,  1,  7,  0,  4,  1,  0,  0]

Up-Sweep Round 1: [3,  4,  7,  7,  4,  5,  0,  0]
                      ↑       ↑       ↑
                    1+3     0+7     1+4

Up-Sweep Round 2: [3,  4,  7, 11,  4,  5,  0,  5]
                          ↑               ↑
                        7+4             0+5

Up-Sweep Round 3: [3,  4,  7, 11,  4,  5,  0, 16]
                                      ↑
                                    5+11

清零根节点:       [3,  4,  7, 11,  4,  5,  0,  0]

Down-Sweep R1:    [3,  4,  7,  0,  4,  5,  0, 11]
                          ↑               ↑
                      temp[3]=0    temp[7]=0+11

Down-Sweep R2:    [3,  0,  7,  4,  4, 11,  0, 16]
                      ↑       ↑       ↑       ↑
                  t[1]=0  t[3]=4  t[5]=11 t[7]=16

Down-Sweep R3:    [0,  3,  4, 11, 11, 15, 16, 16]
                   ↑   ↑   ↑   ↑   ↑   ↑
输出:             [0,  3,  4, 11, 11, 15] (排他型前缀和)
```

**验证**:
- output[0] = 0 (第一个元素前面没有元素)
- output[1] = 3 (3)
- output[2] = 3+1 = 4
- output[3] = 3+1+7 = 11
- output[4] = 3+1+7+0 = 11
- output[5] = 3+1+7+0+4 = 15 ✓

## 线程负载分析

```
N=6, 实际处理 8 个元素, 线程数 = 4

Up-Sweep 各轮活跃线程:
  d=4: Thread 0,1,2,3 工作 (全部)     → 4 个线程
  d=2: Thread 0,1 工作               → 2 个线程
  d=1: Thread 0 工作                 → 1 个线程

Down-Sweep 各轮活跃线程:
  d=1: Thread 0 工作                 → 1 个线程
  d=2: Thread 0,1 工作               → 2 个线程
  d=4: Thread 0,1,2,3 工作 (全部)     → 4 个线程
```

**特点**: 线程负载不均衡，越往树的上层工作的线程越少。但由于工作量本身在减少，总体效率仍然很高。

## 关键代码行详解

### 第 14 行: 动态共享内存
```cpp
extern __shared__ T temp[];
```
- 运行时确定大小，通过内核启动参数传递
- 大小 = n * sizeof(T)

### 第 31-32 行: 索引计算
```cpp
int ai_local = offset * (2 * thid + 1) - 1;
int bi_local = offset * (2 * thid + 2) - 1;
```
- 基于二叉树的数学推导
- offset 每轮翻倍，控制步长

### 第 55-57 行: Down-Sweep 核心
```cpp
T t = temp[ai_local];
temp[ai_local] = temp[bi_local];
temp[bi_local] += t;
```
- 三行代码实现：保存、交换、累加
- ai 获得父节点的值（排他型前缀和）
- bi 获得包含型前缀和

## 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 工作量 (Work) | O(N) | 2N 次操作 (Up-Sweep + Down-Sweep) |
| 跨度 (Span) | O(log N) | 2*log₂N 步并行计算 |
| 共享内存 | N × sizeof(T) | 单缓冲，比 Hillis-Steele 节省 50% |
| 同步次数 | 2 × log₂N | 每轮循环需要 __syncthreads() |

## 与 V1 Hillis-Steele 对比

| 特性 | V1 Hillis-Steele | V2 Blelloch |
|------|-------------------|-------------|
| 工作量 | O(N log N) | **O(N)** ✓ |
| 跨度 | O(log N) | O(log N) |
| 共享内存 | 2N | **N** ✓ |
| 代码复杂度 | 简单 | 中等 |

## 使用限制

1. **n 必须是 2 的幂次** - 如果不是，需要填充到最近的 2 的幂次
2. **线程数限制** - 最大线程数 1024，意味着 n ≤ 2048
3. **单 Block 限制** - 实现只使用 1 个 block，大数据量需要分块处理

---

## 总结

Blelloch 算法的核心优势是**工作量 O(N)**，通过两次遍历二叉树（上归约 + 下传播）实现。虽然代码比 Hillis-Steele 复杂，但效率更高，是生产环境中 GPU 扫描的基础算法。

关键记忆点:
- Up-Sweep: 右节点 += 左节点，构建部分和树
- 根节点清零: 排他型扫描的关键
- Down-Sweep: 交换并累加，ai=父节点值, bi+=ai旧值
