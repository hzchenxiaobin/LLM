🚀 TopK V3: Warp Shuffle

工业级 TopK 实现 - 利用 Warp 级原语实现零共享内存高性能归并

1. 2D 线程组织 (Warp-per-Row)

与 V1 的 Thread-per-Row 不同，V3 使用一个 Warp (32 线程) 协作处理一行数据：

dim3 blockDim(32, 4);  // 32 lanes × 4 warps
row = blockIdx.x * 4 + threadIdx.y  // 由 y 维度决定行

Block 结构可视化

Block 0: 包含 4 个 Warps，每个 Warp 处理一行
🎯 Warp 0 (threadIdx.y=0) → 处理 Row 0
🎯 Warp 1 (threadIdx.y=1) → 处理 Row 1

2. 交错内存访问 (Coalesced Access)

32 个线程交错读取数据，实现完美合并访存：

未访问
Lane 0 访问
Lane 1 访问
Lane 2 访问

内存布局 (N=32):

// 完美合并访存: 一条指令读取 32 个连续 float for (int i = lane_id; i < N; i += 32) { float val = row_input[i]; // 地址连续! }

3. Warp Shuffle 归约

使用 __shfl_down_sync 在寄存器间直接交换数据，无需共享内存：

点击"下一步"查看归约过程
32 个线程比较找出全局最大值

归约树可视化

// 5 轮 shuffle 归约 (log2(32) = 5) for (int offset = 16; offset > 0; offset /= 2) { float other = __shfl_down_sync(0xffffffff, max_val, offset); if (other > max_val) max_val = other; }

6. 核心优化总结

🎯 V3 的三大核心优势

  • 完美合并访存：32 线程交错读取，最大化内存带宽利用率
  • 零共享内存：Warp shuffle 直接在寄存器间交换，无需 __shared__
  • 隐式同步：Warp 内线程天然同步，消除 __syncthreads() 开销
