# TopK V2: Block-per-Row + Shared Memory 版本详解

## 概述

这是 TopK 算法的**协作式并行版本**，采用 **Block-per-Row** 策略：一个 Block 内的所有线程协作处理一行数据，通过共享内存（Shared Memory）实现高效的块内规约。

---

## 核心算法思想（两阶段处理）

### 阶段一：并行 Local Top-K（无同步）

```
输入: Row = [ 3.5, 8.2, 1.4, 9.1, 4.7, 2.3, 7.8, 5.6, 6.9, 0.8 ]  N=10, K=3
       │     │     │     │     │     │     │     │     │     │
       └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
              ↓ 分配给一个 Block 的多个线程 (stride = blockDim.x)

T0 处理: 索引 0, 3, 6   →  values [3.5, 9.1, 7.8]
                          top-3 = [9.1, 7.8, 3.5]  local_vals[0..2]
                          
T1 处理: 索引 1, 4, 7   →  values [8.2, 4.7, 5.6]
                          top-3 = [8.2, 5.6, 4.7]
                          
T2 处理: 索引 2, 5, 8   →  values [1.4, 2.3, 6.9]
                          top-3 = [6.9, 2.3, 1.4]
                          
T3 处理: 索引 9         →  values [0.8]
                          top-3 = [0.8, -∞, -∞]

每个线程维护一个 Local Top-K 数组（寄存器中）
```

**关键代码**:
```cuda
// 协作式遍历：每个线程处理 stride = blockDim.x 的元素
for (int i = tid; i < N; i += blockDim.x) {
    float val = row_input[i];  // 合并访存！
    // 插入排序维护 local top-k
    if (val > local_vals[K - 1]) {
        // ... 插入排序
    }
}
```

### 阶段二：Block 级别规约（树状规约）

```
循环 K 次，每次找全局第 k 大的值

第 1 次迭代 (找第 1 大):
┌──────────────────────────────────────────────────────────────┐
│  Thread:  T0      T1      T2      T3      ...   T254    T255  │
│  local:   9.1     8.2     6.9     0.8     ...   x       y    │
│           │       │       │       │               │       │   │
│           └───────┴───────┴───────┴───────────────┴───────┘   │
│                   Shared Memory (s_vals[256])                 │
│  __syncthreads()                                              │
│                                                               │
│  树状规约 (stride=128 → 64 → 32 → 16 → 8 → 4 → 2 → 1):       │
│                                                               │
│  Step 1 (stride=128):  比较 T0/T128, T1/T129, ...             │
│                        取较大者存入 s_vals[0..127]           │
│                                                               │
│  Step 2 (stride=64):   比较 T0/T64, T1/T65, ...                │
│                        ...                                   │
│                                                               │
│  Step 8 (stride=1):    s_vals[0] = 全局最大值 9.1             │
│                        s_owners[0] = T0 (胜出者)              │
└──────────────────────────────────────────────────────────────┘

T0 写入全局内存: out_vals[0] = 9.1, out_inds[0] = 3

T0 (胜出者) 指针下移: local_ptr = 1, current_val = 7.8 (准备下一轮)
```

```
第 2 次迭代 (找第 2 大):
┌──────────────────────────────────────────────────────────────┐
│  Thread:  T0      T1      T2      T3      ...                 │
│  local:   7.8     8.2     6.9     0.8     ...  ← T0 已更新    │
│           │       │       │       │                            │
│  树状规约结果: s_vals[0] = 8.2 (来自 T1)                      │
│               s_owners[0] = T1                               │
└──────────────────────────────────────────────────────────────┘

T0 写入全局内存: out_vals[1] = 8.2, out_inds[1] = 1

T1 (胜出者) 指针下移: local_ptr = 1, current_val = 5.6
```

```
第 3 次迭代 (找第 3 大):
┌──────────────────────────────────────────────────────────────┐
│  Thread:  T0      T1      T2      T3      ...                 │
│  local:   7.8     5.6     6.9     0.8     ...                  │
│           │       │       │       │                            │
│  树状规约结果: s_vals[0] = 7.8 (来自 T0)                      │
└──────────────────────────────────────────────────────────────┘

最终结果: [9.1, 8.2, 7.8]
```

## 代码逐行解析

### 1. 内核配置（Block-per-Row）

```cuda
int row = blockIdx.x;  // 1 个 Block 处理 1 行
if (row >= Batch) return;
int tid = threadIdx.x;
```

与 V1 的区别:
- V1: `row = blockIdx.x * blockDim.x + threadIdx.x`（线程级并行）
- V2: `row = blockIdx.x`（块级并行）

### 2. 协作式数据加载（合并访存）

```cuda
// 关键：协作式遍历实现合并内存访问
for (int i = tid; i < N; i += blockDim.x) {
    float val = row_input[i];  // 相邻线程访问相邻地址
    // ...
}
```

**合并访存图解**:
```
内存地址: 0x1000  0x1004  0x1008  0x100C  0x1010  0x1014  ...
          │       │       │       │       │       │
          └───────┴───────┴───────┘       └───────┴───────┘
              Warp 0 访问 (合并为 1 个事务)   Warp 1 访问

T0: 读取 i=0   (地址 0x1000)
T1: 读取 i=1   (地址 0x1004)  ← 同 warp 内，合并访问
T2: 读取 i=2   (地址 0x1008)
T3: 读取 i=3   (地址 0x100C)
...继续 stride...
T32: 读取 i=32  (地址 0x1080) ← 新 warp
```

### 3. Shared Memory 规约核心

```cuda
// 定义共享内存数组
__shared__ float s_vals[256];     // 存储当前候选值
__shared__ int s_inds[256];       // 存储对应索引
__shared__ int s_owners[256];     // 记录值属于哪个线程（关键！）
```

**树状规约图解** (假设 8 线程简化):
```
初始: s_vals = [7, 3, 9, 2, 5, 8, 1, 4]

stride = 4:    比较 (0,4) (1,5) (2,6) (3,7)
               结果: [7, 8, 9, 4, -, -, -, -]
               
stride = 2:    比较 (0,2) (1,3)
               结果: [9, 8, -, -, -, -, -, -]
               
stride = 1:    比较 (0,1)
               结果: [9, -, -, -, -, -, -, -] ← 最大值在 s_vals[0]
```

### 4. 胜者指针下移机制

```cuda
// 胜出的线程，指针下移，准备下一轮
int winner = s_owners[0];  // 读取胜出者线程 ID
if (tid == winner) {
    local_ptr++;  // 移动到 Local Top-K 的下一个元素
    if (local_ptr < K) {
        current_val = local_vals[local_ptr];
        current_idx = local_inds[local_ptr];
    } else {
        current_val = -1e20f;  // 已耗尽，标记为无效
    }
}
```

这是 **k-way merge** 的核心：每次找出所有候选中的最大值，然后该候选的提供者贡献下一个候选。

```
初始状态:
┌────┬────┬────┬────┬────┬────┬────┬────┐
│ T0 │ T1 │ T2 │ T3 │ T4 │ T5 │ T6 │ T7 │
├────┼────┼────┼────┼────┼────┼────┼────┤
│ 7  │ 3  │ 9  │ 2  │ 5  │ 8  │ 1  │ 4  │  s_vals[]
│ 0  │ 1  │ 2  │ 3  │ 4  │ 5  │ 6  │ 7  │  s_owners[]
└────┴────┴────┴────┴────┴────┴────┴────┘

┌────┬────┬────┬────┬────┬────┬────┬────┐

└────┴────┴────┴────┴────┴────┴────┴────┘

└────┴────┴────┴────┴────┴────┴────┴────┘

Step 3 (stride = 1): T0 比较 T1
┌────┬────┬────┬────┬────┬────┬────┴────┐
│ 9  │ 8  │ 9  │ 4  │ 5  │ 8  │ 1  │ 4  │  ← s_vals[0]=max(8,9)=9
│ 2  │ 5  │ 2  │ 7  │ 4  │ 5  │ 6  │ 7  │  ← s_owners[0]=2 (胜出者 T2)
└────┴────┴────┴────┴────┴────┴────┴────┘

最终结果: 最大值 9 来自 T2
T2 指针下移，下一轮提供其 Local Top-K 的第 2 个元素
```

## 复杂度分析

### 时间复杂度

**阶段一（Local Top-K）**:
- 每个线程处理约 `N / blockDim.x` 个元素
- 时间: O((N / blockDim.x) × K)

**阶段二（规约）**:
- K 次迭代，每次树状规约 O(log(blockDim.x))
- 时间: O(K × log(blockDim.x))

**总体**: O((N / blockDim.x) × K + K × log(blockDim.x))

当 `blockDim.x = 256` 时:
- 相比 V1 的 O(N × K)，N 维度并行度提高 256 倍！

### 空间复杂度

- **共享内存**: O(blockDim.x) × 3 个数组 (vals, inds, owners)
- **寄存器**: 每个线程 O(K) for local_vals + O(K) for local_inds
- **全局内存**: O(Batch × K) 输出

---

## 优缺点分析

### 优点

1. **协作并行**: Block 内线程协作，充分利用 GPU 计算资源
2. **合并访存**: `i += blockDim.x` 模式确保全局内存合并访问
3. **Shared Memory 高效**: 低延迟规约，避免全局内存同步
4. **可扩展性**: N 越大优势越明显（更多的线程参与计算）

### 缺点

1. **同步开销**: 需要 `__syncthreads()`，可能产生空闲周期
2. **Shared Memory 限制**: 每个 Block 占用 256×3×4=3KB，限制同时运行的 Block 数
3. **线程利用率**: K 次迭代中，每次只有 `s_owners[0]` 线程真正更新状态
4. **复杂度高**: 相比 V1 难以理解，调试更困难

---

## 适用场景

- **N 较大**（如 N > 1000）：协作并行优势明显
- **Batch 适中**：Batch 过大时，V2 每个 Block 处理一行，可能占用过多资源
- **需要高性能**：接受复杂度换取性能
- **硬件共享内存充足**：确保不会成为瓶颈

## 完整调用示例

```cpp
void launch_topk_v2(const float* d_input, float* d_out_vals, int* d_out_inds,
                    int Batch, int N, int K) {
    // 注意：这里是 Batch 个 Block，每个 Block 256 线程
    const int threadsPerBlock = 256;
    const int numBlocks = Batch;  // 1 Block 处理 1 行
    
    topk_v2_kernel<<<numBlocks, threadsPerBlock>>>(
        d_input, d_out_vals, d_out_inds, Batch, N, K
    );
    
    CHECK_CUDA(cudaDeviceSynchronize());
}
```

## V1 vs V2 性能对比

| 指标 | V1 (Thread-per-Row) | V2 (Block-per-Row) | 提升 |
|------|---------------------|-------------------|------|
| 并行度 | Batch 个线程 | Batch × 256 个线程 | 256× |
| 内存访问 | 非合并（每个线程独立） | 完全合并 | 显著 |
| 同步开销 | 无 | 每轮 `__syncthreads()` | 有 |
| 适用 N | 任意 | N > 1000 效果最佳 | - |
| 代码复杂度 | 简单 | 中等 | - |

## 与其他版本对比

| 版本 | 策略 | 时间复杂度 | 特点 |
|------|------|-----------|------|
| V1 | Thread-per-Row | O(N×K) | 简单直观，无协作 |
| **V2** | **Block + Shared Memory** | **O((N/256)×K + K×log(256))** | **协作并行，合并访存** |
| V3 | Warp Shuffle | O((N/32)×K) | Warp 级原语，无需共享内存 |
| V4 | Radix Select | O(N) | 基于位运算，与 K 无关 |
| V5 | Blackwell | - | 新架构特性 |

V2 是**实用性与性能的平衡点**，它展示了 CUDA 协作并行的核心思想，是理解 GPU 编程的关键一步。
