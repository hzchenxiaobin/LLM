💡 关键洞察： 不是一次性排序所有 32×K 个候选，而是逐轮"选举"找最大值。 获胜线程移除已输出的值，下一轮自动找次大值。

📋 版本对比

版本 适用 K 范围 存储 同步开销 核心优势
V3 Warp-per-Row K ≤ 32 纯寄存器 无 __syncthreads() 极致低延迟，生产首选
V4 Radix Select K > 32 寄存器 + Shared Memory Block 同步 支持大K，稳定性好
V2 Block-per-Row 通用 Shared Memory Block 同步 实现简单，合并访存好

💻 核心代码

1. 浮点数键值转换

// 将 float 转为可排序的 uint32 __device__ __inline__ uint32_t float_to_sortable_uint(float val) { uint32_t u = __float_as_uint(val); // 正数：符号位0，转为 1xxx (大值) // 负数：符号位1，按位取反使得 -0.1 > -1.0 return (u & 0x80000000) ? (~u) : (u | 0x80000000); }

2. Warp Shuffle Tree Reduction

// 5轮比较找出32线程中的最大值 #pragma unroll for (int offset = 16; offset > 0; offset /= 2) { float other_val = __shfl_down_sync(0xffffffff, candidate_val, offset); int other_idx = __shfl_down_sync(0xffffffff, candidate_idx, offset); if (other_val > candidate_val) { candidate_val = other_val; candidate_idx = other_idx; } }

3. 两阶段算法

// Phase 1: 粗筛选 - 每个线程收集候选 for (int i = lane_id; i < N; i += 32) { radix_insert_topk(local_vals, local_inds, K, row_input[i], i); } // Phase 2: 精排序 - Warp归约逐轮选举 for (int k = 0; k < K; ++k) { float max_val = warp_reduce_max(...); // 找当前最大 if (lane_id == 0) row_out_vals[k] = max_val; // 输出 if (my_val == max_val) local_ptr++; // 获胜者前进 }

🎯 使用建议

1
K ≤ 32

使用 V4 Warp Kernel，无需 Shared Memory，极致性能

2
K > 32

使用 V4 Block Kernel，Shared Memory 辅助，支持更大 K 值

3
K > 256

考虑多 Pass 策略或 CUB 库的 BlockRadixSort
