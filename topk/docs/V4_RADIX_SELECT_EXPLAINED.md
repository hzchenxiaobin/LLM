# V4: Radix Select (基数选择) 算法详解

## 概述

V4 版本引入了 **Radix Select** 算法——这是工业级 GPU Top-K 实现的核心算法（被 NVIDIA CUB、PyTorch、TensorFlow 等广泛采用）。与 V1-V3 的"维护局部 Top-K 数组"思路完全不同，Radix Select 通过**位级直方图统计**来定位 Top-K 元素，避免了维护有序数组的开销。

---

## 核心思想：不排序，只定位

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Radix Select vs 传统排序                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  传统方法 (V0 - 全局排序):                                                    │
│  ┌───────────────────────────────────────────────────────────────────────┐   │
│  │ 输入: [95, 82, 78, 90, 88, 75, 92, 80] (N=8, K=3)                    │   │
│  │                                                                       │   │
│  │ 步骤: 完全排序 (O(N log N))                                          │   │
│  │   [95, 92, 90, 88, 82, 80, 78, 75]                                    │   │
│  │                                                                       │   │
│  │ 结果: 取前 3 个 [95, 92, 90]                                          │   │
│  │                                                                       │   │
│  │ 缺点: 对所有 N 个元素排序，大部分工作浪费！                            │   │
│  └───────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ════════════════════════════════════════════════════════════════════════  │
│                                                                             │
│  Radix Select (V4):                                                         │
│  ┌───────────────────────────────────────────────────────────────────────┐   │
│  │ 输入: [95, 82, 78, 90, 88, 75, 92, 80] (N=8, K=3)                    │   │
│  │                                                                       │   │
│  │ 二进制表示 (简化 8-bit):                                              │   │
│  │   95 = 10111111                                                       │   │
│  │   92 = 10111000                                                       │   │
│  │   90 = 10110010                                                       │   │
│  │   88 = 10110000                                                       │   │
│  │   82 = 10100100                                                       │   │
│  │   ...                                                                 │   │
│  │                                                                       │   │
│  │ Pass 1: 统计最高 2 位 (10xxxxxx)                                      │   │
│  │   桶 10: 8 个元素 (全部)                                              │   │
│  │   → Top-3 肯定在桶 10 中                                              │   │
│  │                                                                       │   │
│  │ Pass 2: 统计第 3-4 位 (10 11 xxxx = 0xB)                              │   │
│  │   桶 11: 4 个元素 (95, 92, 90, 88)                                    │   │
│  │   → Top-3 肯定在桶 11 (0xB) 中！                                      │   │
│  │                                                                       │   │
│  │ Pass 3: 统计第 5-6 位 (1011 11 xx = 0x3)                              │   │
│  │   桶 11: 1 个元素 (95)  桶 10: 2 个元素 (92, 90)                      │   │
│  │   → 找到 Top-3: [95, 92, 90]                                          │   │
│  │                                                                       │   │
│  │ 复杂度: O(N × P) P=pass 数 (通常 4)，接近 O(N)！                        │   │
│  └───────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  关键洞察: 不需要知道所有元素的精确顺序，只需要定位 Top-K 所在的分区！      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Float 到 UInt 的转换

### 为什么要转换？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    IEEE 754 Float 二进制结构                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  32-bit Float:                                                              │
│  ┌────┬─────────────────────────────────────────────────────────────┐      │
│  │ 31 │ 30-23                    │ 22-0                              │      │
│  │Sign│ Exponent (8 bits)        │ Mantissa (23 bits)                │      │
│  ├────┼──────────────────────────┼───────────────────────────────────┤      │
│  │ 0  │ 10000101                 │ 01111110000000000000000           │      │
│  └────┴─────────────────────────────────────────────────────────────┘      │
│                                                                             │
│  问题: Float 不能直接按位比较大小！                                          │
│                                                                             │
│  示例:                                                                        │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │ 正数: 0x3F800000 = 1.0f                                               │ │
│  │ 负数: 0xBF800000 = -1.0f  (符号位为 1)                                 │ │
│  │                                                                       │ │
│  │ 按 uint 比较: 0x3F800000 < 0xBF800000 (错误!)                          │ │
│  │ 实际值比较:   1.0f > -1.0f (正确)                                      │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│  解决方案: Flip 技巧 (转化为可排序的无符号整数)                              │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │ if (sign bit == 1) {  // 负数                                          │ │
│  │     bits = ~bits;      // 按位取反                                     │ │
│  │ } else {              // 正数                                          │ │
│  │     bits |= 0x80000000; // 设置符号位为 1                             │ │
│  │ }                                                                      │ │
│  │                                                                       │ │
│  │ 转换后顺序:                                                            │ │
│  │   原始:  -∞ ... -1.0f ... 0 ... 1.0f ... +∞                            │ │
│  │   转换:   0 ... 0x7F7FFFFF ... 0x80000000 ... 0xFFFFFFFF               │ │
│  │                                                                       │ │
│  │ 结果: 数值越大，uint 值越大，可以直接按位比较！                          │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 转换代码

```cpp
__device__ __forceinline__ unsigned int float_to_sortable_uint(float value) {
    unsigned int bits;
    memcpy(&bits, &value, sizeof(float));
    
    // Flip 技巧：使 float 可以按 uint 排序
    if (bits & 0x80000000) {      // 负数：符号位为 1
        bits = ~bits;              // 按位取反
    } else {                      // 正数：符号位为 0
        bits |= 0x80000000;        // 设置符号位为 1
    }
    return bits;
}
```

## 算法流程详解

### 总体流程

```

│                    Radix Select 4-Pass 流程                                   │
├─────────────────────────────────────────────────────────────────────────────┤

│  │ 输入: N 个 float 元素                                               │   │
│  │                                                                       │   │
│  │ 1. 每个 Block 计算局部直方图 (256 桶)                                 │   │
│  │    ┌─────────┬─────────┬─────────┬─────────┐                         │   │

│  │ 2. 合并所有 Block 的直方图 → 全局直方图                               │   │

│  │    │ Bucket 255: 100 elements                │                       │   │

│  │    │ Bucket 253: 150 elements (累计 450)     │                       │   │
│  │    │ ...                                     │                       │   │
│  │    └─────────────────────────────────────────┘                       │   │
│  │                                                                       │   │
│  │ 4. 记录阈值: threshold = 254 << 24                                    │   │
│  │    下一轮只处理 Bucket 254 中的元素                                   │   │

│                                                                             │
│  ════════════════════════════════════════════════════════════════════════  │
```

## 代码架构详解

### 1. 直方图计算内核: `radix_histogram_kernel`

```cpp
template <int BLOCK_SIZE>
__global__ void radix_histogram_kernel(const float* input, int N,
                                       unsigned int* histograms,
                                       int bit_start,
                                       unsigned int current_threshold) {
    __shared__ unsigned int s_histogram[RADIX_BUCKETS];  // 256 桶

    int tid = threadIdx.x;
    int gid = tid + blockIdx.x * blockDim.x;
    int stride = blockDim.x * gridDim.x;

    // 初始化共享内存直方图
    for (int i = tid; i < RADIX_BUCKETS; i += blockDim.x) {
        s_histogram[i] = 0;
    }
    __syncthreads();

    // 处理输入数据
    for (int idx = gid; idx < N; idx += stride) {
        float val = input[idx];
        unsigned int bits = float_to_sortable_uint(val);

        // 阈值检查：只处理与当前阈值高 bits 匹配的元素
        bool include = true;
        if (bit_start < 24) {  // 不是最后 8 位
            unsigned int high_bits = bits >> (bit_start + RADIX_BITS);
            unsigned int threshold_high = current_threshold >> RADIX_BITS;
            include = (high_bits == threshold_high);
        }

        if (include) {
            // 提取当前 pass 的 8 位作为桶索引
            unsigned int key = (bits >> bit_start) & RADIX_MASK;  // 0xFF
            atomicAdd(&s_histogram[key], 1);  // 原子累加
        }
    }
    __syncthreads();

    // 写入全局内存 (每个 Block 一个直方图)
    unsigned int* block_hist = histograms + blockIdx.x * RADIX_BUCKETS;
    for (int i = tid; i < RADIX_BUCKETS; i += blockDim.x) {
        block_hist[i] = s_histogram[i];
    }
}
```

### 2. 直方图合并内核: `merge_histograms_kernel`

```cpp
__global__ void merge_histograms_kernel(const unsigned int* block_histograms,
                                        unsigned int* global_histogram,
                                        int num_blocks) {
    int tid = threadIdx.x;

    // 每个线程处理一个桶
    for (int bucket = tid; bucket < RADIX_BUCKETS; bucket += blockDim.x) {
        unsigned int sum = 0;
        // 累加所有 Block 的该桶计数
        for (int b = 0; b < num_blocks; b++) {
            sum += block_histograms[b * RADIX_BUCKETS + bucket];
        }
        global_histogram[bucket] = sum;
    }
}
```

### 3. 候选元素收集内核: `collect_candidates_kernel`

```cpp
__global__ void collect_candidates_kernel(const float* input, int N,
                                            TopKNode* candidates,
                                            int* candidate_count,
                                            unsigned int threshold_bits,
                                            int num_bits_checked) {
    int tid = threadIdx.x;
    int gid = tid + blockIdx.x * blockDim.x;
    int stride = blockDim.x * gridDim.x;

    for (int idx = gid; idx < N; idx += stride) {
        float val = input[idx];
        unsigned int bits = float_to_sortable_uint(val);

        // 检查前 num_bits_checked 位是否匹配阈值
        unsigned int checked_bits = bits >> (32 - num_bits_checked);
        unsigned int threshold_checked = threshold_bits >> (32 - num_bits_checked);

        if (checked_bits == threshold_checked) {
            // 原子获取位置并写入
            int pos = atomicAdd(candidate_count, 1);
            if (pos < N) {
                candidates[pos].value = val;
                candidates[pos].index = idx;
            }
        }
    }
}
```

### 4. 最终选择内核: `final_select_topk_kernel`

```cpp
template <int K>
__global__ void final_select_topk_kernel(TopKNode* candidates,
                                        int candidate_count,
                                        TopKNode* output) {
    extern __shared__ TopKNode smem[];
    int tid = threadIdx.x;

    // 初始化局部 Top-K
    TopKNode local_top[K];
    for (int i = 0; i < K; i++) {
        local_top[i].value = -1e20f;
        local_top[i].index = -1;
    }

    // 处理候选元素
    for (int idx = tid; idx < candidate_count; idx += blockDim.x) {
        float val = candidates[idx].value;
        int index = candidates[idx].index;
        dev_insert_topk(local_top, K, val, index);
    }

    // 共享内存树形归约 (同 V2)
    for (int i = 0; i < K; i++) {
        smem[tid * K + i] = local_top[i];
    }
    __syncthreads();

    for (int s = blockDim.x / 2; s > 0; s >>= 1) {
        // ... 归约逻辑
    }

    // 输出结果
    if (tid == 0) {
        for (int i = 0; i < K; i++) {
            output[i] = smem[i];
        }
    }
}
```

```

│                    V4 Radix Select 完整流程                                  │

│  │          ↑ 高 2 位                                                   │   │
│  │                                                                       │   │
│  │ 直方图:                                                               │   │
│  │   桶 10 (0x2): 8 个元素 (所有元素高 2 位都是 10)                      │   │
│  │                                                                       │   │
│  │ 结论: Top-3 在桶 10 中                                               │   │
│  │ threshold = 0x2 << 6 = 0x80                                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  Pass 2 (bits 5-4):                                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 只考虑桶 10 的元素                                                    │   │
│  │ 二进制: 1011|1111, 1010|0010, 1010|0110, ...                          │   │
│  │             ↑ bits 5-4                                               │   │
│  │                                                                       │   │
│  │ 直方图:                                                               │   │
│  │   桶 11 (0x3): 1 个元素 (95)                                          │   │
│  │   桶 10 (0x2): 4 个元素 (92, 90, 88, 82)                              │   │
│  │   桶 01 (0x1): 3 个元素 (78, 75, 80)                                  │   │
│  │                                                                       │   │
│  │ 从最高桶累加: 桶 11 = 1 < 3, 桶 11+桶 10 = 5 ≥ 3                     │   │
│  │ 结论: Top-3 分布在桶 11 (1个) 和 桶 10 (2个)                        │   │
│  │ threshold = 0x2 << 4 = 0x80 (记录我们需要桶 10 及更高)                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  Pass 3-4: 继续细分直到精确定位                                            │
│                                                                             │
│  最终结果: threshold 精确确定 Top-3 = [95, 92, 90]                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 与 V1-V3 的对比

| 特性 | V1-V3 (局部 Top-K) | V4 (Radix Select) |
|------|-------------------|-------------------|
| **核心思想** | 维护有序数组 | 位级直方图统计 |
| **时间复杂度** | O(N × K) | O(N × P) P=4 passes |
| **空间复杂度** | O(threads × K) | O(N) 候选数组 |
| **大 K 表现** | 寄存器溢出 | 稳定 |
| **分支预测** | 较多 (插入排序) | 极少 (直方图) |
| **适用场景** | K ≤ 64 | K ≤ 1024+ |
| **工业应用** | 教学 | NVIDIA CUB, PyTorch |

---

## 关键优化点

### 1. 分支消除

```
V1-V3: if (val > local_top[K-1]) { while (...) { ... } }  // 大量分支
V4: atomicAdd(&s_histogram[key], 1);  // 无分支，统一执行
```

### 2. 内存访问模式

```
V1-V3: 随机访问 (插入排序导致)
V4: 顺序扫描 + 共享内存原子操作 (可预测，缓存友好)
```

### 3. 可扩展性

```
K=8:   V3 更快 (寄存器优化更好)
K=64:  V4 接近 V3
K=256: V4 明显优于 V3 (V3 寄存器溢出)
K=1024: V4 是唯一可行方案
```

---

## 总结

Radix Select 是**工业级 Top-K 算法**的核心，优势在于：

1. **无分支**: 直方图统计替代插入排序
2. **可扩展**: 不受寄存器限制，支持大 K 值
3. **确定性**: O(N × P) 复杂度，P 为常数 (通常 4)
4. **硬件友好**: 顺序扫描，缓存友好，原子操作利用 GPU 硬件特性

### 适用场景

- ✅ 大 K 值 (K > 64)
- ✅ 需要确定性性能的场景
- ✅ 生产环境 (NVIDIA CUB 同款算法)
- ✅ 理解 GPU 并行直方图算法的优秀材料
