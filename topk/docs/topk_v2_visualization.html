Two-Phase Algorithm

1
Local Top-K Computation
Each thread processes with stride pattern
Coalesced Memory Access Pattern
Threads access memory with stride = blockDim.x (8). Adjacent threads read adjacent memory locations simultaneously.
Phase 1: Local Top-K
// Each thread maintains local Top-K float local_vals[MAX_K]; int local_inds[MAX_K]; // Coalesced stride access for (int i = tid; i < N; i += blockDim.x) { float val = row_input[i]; if (val > local_vals[K - 1]) { // Insertion sort into local array int p = K - 1; while (p > 0 && val > local_vals[p - 1]) { local_vals[p] = local_vals[p - 1]; local_inds[p] = local_inds[p - 1]; p--; } local_vals[p] = val; local_inds[p] = i; } }

Tree Reduction Process
Stride halves each step: 4 → 2 → 1. Thread 0 writes winner to output. Winner thread advances its pointer for next iteration.
Phase 2: Tree Reduction
// Load to shared memory s_vals[tid] = current_val; s_inds[tid] = current_idx; s_owners[tid] = tid; __syncthreads(); // Tree reduction: stride = 4 → 2 → 1 for (int stride = blockDim.x / 2; stride > 0; stride >>= 1) { if (tid < stride) { if (s_vals[tid + stride] > s_vals[tid]) { s_vals[tid] = s_vals[tid + stride]; s_inds[tid] = s_inds[tid + stride]; s_owners[tid] = s_owners[tid + stride]; } } __syncthreads(); } // Winner advances pointer if (tid == s_owners[0]) { local_ptr++; current_val = local_vals[local_ptr]; }

Memory Access Pattern

❌ Non-Coalesced Access (V1 Pattern)
Strided Access Pattern
Each thread reads consecutive elements. Memory requests are serialized, causing poor bandwidth utilization.
✅ Coalesced Access (V2 Pattern)
i += blockDim.x Pattern
Adjacent threads access adjacent memory (tid, tid+8, tid+16...). Combined into single memory transaction. 2x bandwidth!
Memory Access Comparison
// V1: Thread 0 reads [0,1,2,3...], Thread 1 reads [N,N+1,N+2...]
for (int i = 0; i < N; i++) { /* Non-coalesced */ }
// V2: All threads read consecutive elements simultaneously
for (int i = tid; i < N; i += blockDim.x) { /* Coalesced! */ }
