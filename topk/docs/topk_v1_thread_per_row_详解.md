# TopK V1: 单线程串行版本详解

## 概述

这是 TopK 算法的**朴素实现版本**，采用最简单的策略：使用**单个线程**串行处理整个一维数组，找出 Top K 个最大值。

**特点**：
- 1D 输入（无 Batch 轴）
- 只输出 Top-K 值，不输出索引
- 单线程串行执行

---

## 算法核心思想

```
┌─────────────────────────────────────────────────────────┐
│                    输入数据 (1D数组)                     │
│                   长度 N 的 float 数组                    │
├─────────────────────────────────────────────────────────┤
│  [ 3.5, 1.2, 7.8, 2.1, 9.0, 4.5, 6.2, 8.1, ... ]       │
│     0     1     2     3     4     5     6     7         │
└─────────────────────────────────────────────────────────┘
                          ↓ 单线程串行处理
┌─────────────────────────────────────────────────────────┐
│                    Top K 结果                            │
│                   长度 K 的值数组                         │
├─────────────────────────────────────────────────────────┤
│  最大的 K 个值 (按降序排列)                               │
│  例如: [9.0, 8.1, 7.8, ...]                               │
└─────────────────────────────────────────────────────────┘
```

---

## CUDA 线程组织

```
线程配置: 1 block, 1 thread (grid = 1, block = 1)

Grid:  ┌─────┐
       │Block│
       │  0  │
       └──┬──┘
          │
Block: ┌──┴──┐
       │ T0  │  ← 只有线程0工作，其他线程直接返回
       │ 1行 │
       └─────┘

其他线程 (threadIdx.x > 0): 直接 return
```

---

## 代码逐行解析

### 1. 内核函数签名

```cuda
__global__ void topk_v1_kernel(
    const float* input,    // 输入数据: [N]
    float* out_vals,       // 输出值: [K]
    int N,                 // 数组长度
    int K                  // 要找的 Top K 个数
)
```

**注意**：
- 无 `Batch` 参数（1D输入）
- 无 `out_inds` 参数（只输出值，不输出索引）

### 2. 线程过滤

```cuda
// 只使用一个线程处理（线程0）
if (threadIdx.x != 0 || blockIdx.x != 0) return;
```
- 只有 thread 0 of block 0 执行实际计算
- 其他线程直接返回，不做任何工作

### 3. 初始化 Top K 数组

```cuda
float top_vals[MAX_K];   // 存储当前找到的 Top K 值

// 初始化为极小值
for (int i = 0; i < K; ++i) {
    top_vals[i] = -1e20f;  // 负无穷大
}
```
- 使用**寄存器数组**（单个线程私有）
- `MAX_K` 是编译时常量（默认为 32）
- 初始化为极小值确保任何输入值都比它大

### 4. 串行插入排序核心逻辑

```cuda
for (int i = 0; i < N; ++i) {
    float val = input[i];
    
    // 步骤 1: 检查是否大于当前最小值
    if (val > top_vals[K - 1]) {
        int pos = K - 1;
        
        // 步骤 2: 找到插入位置（从后向前移动）
        while (pos > 0 && val > top_vals[pos - 1]) {
            top_vals[pos] = top_vals[pos - 1];  // 值后移
            pos--;
        }
        
        // 步骤 3: 插入新值
        top_vals[pos] = val;
        // 注意：不存储索引
    }
}
```

### 5. 写回结果

```cuda
for (int i = 0; i < K; ++i) {
    out_vals[i] = top_vals[i];
}
```
- 只写入值，不写入索引

## 插入排序工作原理图解

假设 K = 3，输入序列为 [5, 2, 8, 1, 9]

```
初始状态:                    top_vals = [-∞, -∞, -∞]

读取 5:   5 > -∞? 是
          ┌─────────────────────────┐
          │  比较: 5 vs -∞          │
          │  5 > -∞, 移动到 pos=0   │
          └─────────────────────────┘
                            top_vals = [5, -∞, -∞]

读取 2:   2 > -∞? 是
          ┌─────────────────────────┐
          │  比较: 2 vs 5           │
          │  2 < 5, 停在 pos=1      │
          └─────────────────────────┘
                            top_vals = [5, 2, -∞]

读取 8:   8 > -∞? 是
          ┌─────────────────────────┐
          │  比较: 8 vs 2           │  8 > 2, 移动
          │  比较: 8 vs 5           │  8 > 5, 移动
          │  停在 pos=0             │
          └─────────────────────────┘
                            top_vals = [8, 5, 2]

读取 1:   1 > 2? 否
          ┌─────────────────────────┐
          │  1 不够大，忽略          │
          └─────────────────────────┘
                            top_vals = [8, 5, 2]  (不变)

读取 9:   9 > 2? 是
          ┌─────────────────────────┐
          │  比较: 9 vs 2           │  9 > 2, 移动
          │  比较: 9 vs 5           │  9 > 5, 移动
          │  比较: 9 vs 8           │  9 > 8, 移动
          │  停在 pos=0             │
          └─────────────────────────┘
                            top_vals = [9, 8, 5]

最终结果: Top 3 = [9, 8, 5]
```

## 复杂度分析

### 时间复杂度
- **单个线程**: O(N × K)
  - 遍历 N 个元素
  - 每个有效元素最多移动 K 次

### 空间复杂度
- **寄存器**: O(K) 的空间
- **输出**: O(K) 的全局内存

### 内存访问模式
```
输入读取: 顺序访问 (input[i], i=0..N-1)
         ↓ 顺序访问友好
输出写入: 顺序访问 (out_vals[i])
         ↓ 顺序访问友好
```

---

## 优缺点分析

### 优点
1. **实现简单**: 代码直观易懂，易于调试
2. **无同步开销**: 单线程执行，无需任何同步
3. **无共享内存**: 不使用 `__shared__`，简化编程
4. **可预测**: 执行时间固定，无线程间竞争

### 缺点
1. **无并行**: 只使用 1 个线程，完全浪费 GPU 并行能力
2. **寄存器压力**: 大 K 值会增加寄存器使用
3. **性能差**: 仅适用于小数据量调试

---

## 适用场景

- **N 较小**（如 < 1000）：串行遍历不会太长
- **K 较小**（如 K <= 32）：适合寄存器存储
- **快速原型**: 作为基准实现验证正确性
- **调试**: 用于验证其他并行版本的正确性

## 完整调用示例

```cpp
// 主机端代码
void launch_topk_v1(const float* d_input, float* d_out_vals,
                    int N, int K) {
    // V1 只使用 1 个 block, 1 个 thread
    topk_v1_kernel<<<1, 1>>>(d_input, d_out_vals, N, K);
    
    CHECK_CUDA(cudaDeviceSynchronize());
}
```

## 与其他版本对比

| 版本 | 策略 | 时间复杂度 | 特点 | 输出 |
|------|------|-----------|------|------|
| **V1** | 单线程串行 | O(N×K) | 简单直观，无协作 | 只输出值 |
| V2 | Block级协作 | O(N×K/block_size) | 块内协作，减少线程数 | 只输出值 |
| V3 | Warp Shuffle | O(N×K) | Warp级原语，无共享内存 | 只输出值 |
| V4 | Radix Select | O(N) | 基于位运算，与 K 无关 | 只输出值 |

V1 是理解 TopK CUDA 实现的**最佳入门版本**，它清晰展示了基本算法，是后续优化版本的基础。
