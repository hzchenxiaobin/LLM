# TopK V1: Thread-per-Row 版本详解

## 概述

这是 TopK 算法的**朴素实现版本**，采用最简单的并行策略：每个线程处理一行数据，串行地在该行中找出 Top K 个最大值。

---

## 算法核心思想

```
┌─────────────────────────────────────────────────────────┐
│                    输入数据矩阵                          │
│              (Batch 行 × N 列的 float 数组)               │
├─────────────────────────────────────────────────────────┤
│  Row 0: [ 3.5, 1.2, 7.8, 2.1, 9.0, 4.5, ... ]           │
│  Row 1: [ 6.2, 8.1, 2.3, 5.7, 1.9, 3.4, ... ]           │
│  Row 2: [ 4.1, 9.5, 3.2, 7.6, 2.8, 6.3, ... ]           │
│     ...                                                 │
│  Row Batch-1: [ ... ]                                   │
└─────────────────────────────────────────────────────────┘
                          ↓ 每个线程处理一行
┌─────────────────────────────────────────────────────────┐
│                    Top K 结果                            │
│              (Batch 行 × K 列)                           │
├─────────────────────────────────────────────────────────┤
│  Row 0: 最大的 K 个值及其索引                             │
│  Row 1: 最大的 K 个值及其索引                             │
│  Row 2: 最大的 K 个值及其索引                             │
└─────────────────────────────────────────────────────────┘
```

---

## CUDA 线程组织

```
线程网格配置 (1D Grid / 1D Block)

Grid:  ┌─────┬─────┬─────┬─────┬─────┐
       │Block│Block│Block│ ... │Block│
       │  0  │  1  │  2  │     │  M  │
       └─────┴─────┴─────┴─────┴─────┘
              ↓ 每个 Block 包含多个 threads
Block: ┌────┬────┬────┬────┬────┐
       │T0  │T1  │T2  │... │Tn  │  ← n = blockDim.x
       │Row0│Row1│Row2│    │    │
       └────┴────┴────┴────┴────┘

线程索引计算:
    row = blockIdx.x * blockDim.x + threadIdx.x
```

## 代码逐行解析

### 1. 内核函数签名

```cuda
__global__ void topk_v1_kernel(
    const float* input,    // 输入数据: [Batch × N]
    float* out_vals,       // 输出值: [Batch × K]
    int* out_inds,         // 输出索引: [Batch × K]
    int Batch,             // 行数
    int N,                 // 每行元素数
    int K                  // 要找的 Top K 个数
)
```

### 2. 线程索引计算

```cuda
int row = blockIdx.x * blockDim.x + threadIdx.x;
if (row >= Batch) return;
```
- 计算当前线程负责处理的行号
- 边界检查：超过 Batch 的行直接返回

### 3. 数据指针定位

```cuda
const float* row_input = input + row * N;      // 当前行输入起始地址
float* row_out_vals = out_vals + row * K;       // 当前行输出值起始地址
int* row_out_inds = out_inds + row * K;         // 当前行输出索引起始地址
```

### 4. 初始化 Top K 数组

```cuda
float top_vals[MAX_K];   // 存储当前找到的 Top K 值
int top_inds[MAX_K];     // 存储对应值的索引

// 初始化为极小值
for (int i = 0; i < K; ++i) {
    top_vals[i] = -1e20f;  // 负无穷大
    top_inds[i] = -1;       // 无效索引
}
```
- 使用**寄存器数组**（每个线程私有）
- `MAX_K` 是编译时常量（默认为 32）
- 初始化为极小值确保任何输入值都比它大

### 5. 串行插入排序核心逻辑

```cuda
for (int i = 0; i < N; ++i) {
    float val = row_input[i];
    
    // 步骤 1: 检查是否大于当前最小值
    if (val > top_vals[K - 1]) {
        int pos = K - 1;
        
        // 步骤 2: 找到插入位置（从后向前移动）
        while (pos > 0 && val > top_vals[pos - 1]) {
            top_vals[pos] = top_vals[pos - 1];  // 值后移
            top_inds[pos] = top_inds[pos - 1];   // 索引后移
            pos--;
        }
        
        // 步骤 3: 插入新值
        top_vals[pos] = val;
        top_inds[pos] = i;
    }
}
```

### 6. 写回结果

```cuda
for (int i = 0; i < K; ++i) {
    row_out_vals[i] = top_vals[i];
    row_out_inds[i] = top_inds[i];
}
```

## 插入排序工作原理图解

假设 K = 3，输入序列为 [5, 2, 8, 1, 9]

```
初始状态:                    top_vals = [-∞, -∞, -∞]
                            top_inds = [-1, -1, -1]

读取 5:   5 > -∞? 是
          ┌─────────────────────────┐
          │  比较: 5 vs -∞          │
          │  5 > -∞, 移动到 pos=0   │
          └─────────────────────────┘
                            top_vals = [5, -∞, -∞]
                            top_inds = [0, -1, -1]

读取 2:   2 > -∞? 是
          ┌─────────────────────────┐
          │  比较: 2 vs 5           │
          │  2 < 5, 停在 pos=1      │
          └─────────────────────────┘
                            top_vals = [5, 2, -∞]
                            top_inds = [0, 1, -1]

读取 8:   8 > -∞? 是
          ┌─────────────────────────┐
          │  比较: 8 vs 2           │  8 > 2, 移动
          │  比较: 8 vs 5           │  8 > 5, 移动
          │  停在 pos=0             │
          └─────────────────────────┘
                            top_vals = [8, 5, 2]
                            top_inds = [2, 0, 1]

读取 1:   1 > 2? 否
          ┌─────────────────────────┐
          │  1 不够大，忽略          │
          └─────────────────────────┘
                            top_vals = [8, 5, 2]  (不变)

读取 9:   9 > 2? 是
          ┌─────────────────────────┐
          │  比较: 9 vs 2           │  9 > 2, 移动
          │  比较: 9 vs 5           │  9 > 5, 移动
          │  比较: 9 vs 8           │  9 > 8, 移动
          │  停在 pos=0             │
          └─────────────────────────┘
                            top_vals = [9, 8, 5]
                            top_inds = [4, 2, 0]

最终结果: Top 3 = [9, 8, 5], 索引 = [4, 2, 0]
```

## 复杂度分析

### 时间复杂度
- **每个线程**: O(N × K)
  - 遍历 N 个元素
  - 每个有效元素最多移动 K 次
- **总体**: O(Batch × N × K)

### 空间复杂度
- **每个线程**: O(K) 的寄存器空间
- **总体**: O(Batch × K) 的输出空间

### 内存访问模式
```
输入读取: 顺序访问 (input[row * N + i], i=0..N-1)
         ↓ 合并访问友好，但每个线程独立
输出写入: 顺序访问 (out_vals[row * K + i])
         ↓ 合并访问友好
```

---

## 优缺点分析

### 优点
1. **实现简单**: 代码直观易懂，易于调试
2. **线程独立**: 无同步开销，无线程间竞争
3. **无共享内存**: 不使用 __shared__，简化编程
4. **可预测**: 每个线程工作量固定

### 缺点
1. **线程利用率低**: 每个线程只做 O(N×K) 工作，可能不足以掩盖延迟
2. **寄存器压力**: 大 K 值会增加寄存器使用
3. **无协作**: 线程之间完全不协作，浪费 GPU 并行能力
4. **扩展性差**: 仅依赖线程级并行，未利用 warp 级并行

---

## 适用场景

- **Batch 较大**（如 > 10000）：可以充分利用 GPU 多线程
- **N 较小**（如 < 1000）：串行遍历不会太长
- **K 较小**（如 K <= 32）：适合寄存器存储
- **快速原型**: 作为基准实现验证正确性

## 完整调用示例

```cpp
// 主机端代码
void launch_topk_v1(const float* d_input, float* d_out_vals, int* d_out_inds,
                    int Batch, int N, int K) {
    // 配置: 每 block 256 线程，刚好填满一个 warp
    const int threadsPerBlock = 256;
    const int blocksPerGrid = (Batch + threadsPerBlock - 1) / threadsPerBlock;
    
    topk_v1_kernel<<<blocksPerGrid, threadsPerBlock>>>(
        d_input, d_out_vals, d_out_inds, Batch, N, K
    );
    
    CHECK_CUDA(cudaDeviceSynchronize());
}
```

---

## 性能提示

1. **块大小选择**: 256 是较好的默认值，但可以通过实验调整
2. **K 值限制**: K > 32 时寄存器压力增大，可能需要共享内存版本
3. **内存布局**: 确保输入数据是行优先（row-major）布局
4. **边界处理**: Batch 不是 block 大小整数倍时，边界线程会空闲

## 与其他版本对比

| 版本 | 策略 | 时间复杂度 | 特点 |
|------|------|-----------|------|
| **V1** | Thread-per-Row | O(N×K) | 简单直观，无协作 |
| V2 | Shared Memory | O(N×K/block_size) | 块内协作，减少线程数 |
| V3 | Warp Shuffle | O(N×K) | Warp 级原语，无共享内存 |
| V4 | Radix Select | O(N) | 基于位运算，与 K 无关 |
| V5 | Blackwell | - | 利用新架构特性 |

V1 是理解 TopK CUDA 实现的**最佳入门版本**，它清晰展示了基本算法，是后续优化版本的基础。
