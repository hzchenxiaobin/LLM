## 整体架构

V1 采用经典的 **Map-Reduce** 架构：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         V1 架构概览                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Input Array (N elements)                                            │
│  ┌────────────────────────────────────────┐                         │
│  │  [0] [1] [2] ... [N-2] [N-1]          │                         │
│  └────────────────────────────────────────┘                         │
│       │      │      │         │      │                              │
│       ▼      ▼      ▼         ▼      ▼                              │
│  ┌────────────────────────────────────────┐                         │
│  │   Map Phase (topk_v1_kernel)           │                         │
│  │   ┌─────┐ ┌─────┐        ┌─────┐     │                         │
│  │   │ T0  │ │ T1  │  ...   │ Tm  │     │  m = num_blocks ×       │
│  │   │[K]  │ │[K]  │        │[K]  │     │     num_threads         │
│  │   └─────┘ └─────┘        └─────┘     │                         │
│  │   Each thread maintains local Top-K   │                         │
│  └────────────────────────────────────────┘                         │
│       │      │      │         │      │                              │
│       ▼      ▼      ▼         ▼      ▼                              │
│  ┌────────────────────────────────────────┐                         │
│  │   Global Memory                        │                         │
│  │   thread_tops[m][K]                    │                         │
│  └────────────────────────────────────────┘                         │
│                     │                                                │
│                     ▼                                                │
│  ┌────────────────────────────────────────┐                         │
│  │   Reduce Phase (topk_v1_reduce_kernel) │                         │
│  │   ┌──────────────────────────────┐    │                         │
│  │   │  Tree Reduction in SMEM    │    │                         │
│  │   │  pairs → merge → top K       │    │                         │
│  │   └──────────────────────────────┘    │                         │
│  └────────────────────────────────────────┘                         │
│                     │                                                │
│                     ▼                                                │
│  ┌────────────────────────────────────────┐                         │
│  │   Output: Top K elements             │                         │
│  └────────────────────────────────────────┘                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 核心概念

### 1. Grid-Stride Loop

```
传统方式 (每个线程处理1个元素)          Grid-Stride (每个线程处理多个元素)
┌─────────────────────┐                 ┌─────────────────────────────────────┐
│  Thread 0: [0]      │                 │  Thread 0: [0] [256] [512] ...      │
│  Thread 1: [1]      │                 │  Thread 1: [1] [257] [513] ...      │
│  ...                │                 │  ...                                │
│  Thread 255: [255]  │                 │  Thread 255: [255] [511] [767] ...  │
└─────────────────────┘                 └─────────────────────────────────────┘

stride = blockDim.x × gridDim.x = 256
for (int idx = tid; idx < N; idx += stride)
```

### 2. 局部 Top-K 维护

每个线程在寄存器中维护一个大小为 K 的有序数组：

```
Thread 0's local_top[K=4]:
┌────────────────────────────────────────┐
│ Index │  0   │  1   │  2   │  3   │   │
├────────────────────────────────────────┤
│ Value │ 95.2 │ 87.1 │ 62.3 │ 45.8 │ ←─┼── 最小值 threshold
│       │      │      │      │      │   │
│       │  ↓   │      │      │      │   │
│       │ 新值 88.5 > 45.8? 是!     │   │
│       │      │      │      │      │   │
│       │  0   │  1   │  2   │  3   │   │
│       │ 95.2 │ 88.5 │ 87.1 │ 62.3 │ ←─┘── 插入并移位
└────────────────────────────────────────┘

插入逻辑:
1. 比较新值与 local_top[K-1] (当前最小值)
2. 如果更大，从末尾开始查找插入位置
3. 移位并插入，保持降序
```

## Kernel 1: topk_v1_kernel

### 代码结构

```cpp
template <int K>
__global__ void topk_v1_kernel(const float* input, int N, TopKNode* thread_tops)
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `input` | `const float*` | 输入数组 (设备内存) |
| `N` | `int` | 输入元素个数 |
| `thread_tops` | `TopKNode*` | 输出缓冲区 [num_threads][K] |

### 执行流程

```
┌────────────────────────────────────────────────────────────────┐
│                    topk_v1_kernel 执行流程                      │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 计算全局线程 ID                                             │
│     ┌─────────────────────────────────────┐                    │
│     │ tid = blockIdx.x × blockDim.x     │                    │
│     │       + threadIdx.x               │                    │
│     └─────────────────────────────────────┘                    │
│                          │                                      │
│                          ▼                                      │
│  2. 初始化局部 Top-K (寄存器)                                   │
│     ┌─────────────────────────────────────┐                    │
│     │ for i = 0 to K-1:                   │                    │
│     │   local_top[i].value = -1e20f       │                    │
│     │   local_top[i].index = -1           │                    │
│     └─────────────────────────────────────┘                    │
│                          │                                      │
│                          ▼                                      │
│  3. Grid-Stride 循环处理数据                                    │
│     ┌─────────────────────────────────────┐                    │
│     │ for (idx = tid; idx < N;          │                    │
│     │      idx += stride) {             │                    │
│     │   val = input[idx]                │                    │
│     │   if (val > local_top[K-1])       │                    │
│     │     insert_and_shift(val, idx)    │                    │
│     │ }                                 │                    │
│     └─────────────────────────────────────┘                    │
│                          │                                      │
│                          ▼                                      │
│  4. 写回全局内存                                                │
│     ┌─────────────────────────────────────┐                    │
│     │ out_offset = tid × K              │                    │
│     │ for i = 0 to K-1:                 │                    │
│     │   thread_tops[out_offset + i]     │                    │
│     │     = local_top[i]                │                    │
│     └─────────────────────────────────────┘                    │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 内存布局

```
输入数组 (input):
┌─────────────────────────────────────────────────┐
│ [0] [1] [2] [3] ... [N-3] [N-2] [N-1]         │
└─────────────────────────────────────────────────┘

输出数组 (thread_tops):
┌─────────────────────────────────────────────────────────────────────┐
│ Thread 0    │ Thread 1    │ ... │ Thread m-1                        │
├─────────────────────────────────────────────────────────────────────┤
│ [0]...[K-1] │ [K]...[2K-1]│ ... │ [(m-1)K]...[mK-1]                 │
│  Top-K #0   │  Top-K #1   │     │  Top-K #m-1                       │
│  (降序)     │  (降序)     │     │  (降序)                          │
└─────────────────────────────────────────────────────────────────────┘
         ↑ m = num_blocks × num_threads
         ↑ Layout: [num_threads][K]
```

## Kernel 2: topk_v1_reduce_kernel

### 代码结构

```cpp
template <int K>
__global__ void topk_v1_reduce_kernel(const TopKNode* thread_tops, 
                                      int num_threads,
                                      TopKNode* output)
```

### 树形归约 (Tree Reduction)

```
树形归约过程 (假设 8 个线程，K=2):

初始状态 (Shared Memory):
┌─────────────────────────────────────────────────────────┐
│  SMEM[0]  │ SMEM[1]  │ SMEM[2]  │ ... │ SMEM[7]        │
│ Thread 0  │ Thread 1 │ Thread 2 │     │ Thread 7       │
│ [95, 87]  │ [92, 85] │ [88, 82] │     │ [70, 65]       │
└─────────────────────────────────────────────────────────┘

Iteration 1 (stride = 4):
┌─────────────────────────────────────────────────────────┐
│  T0 merge(T0,T4)  │  T1 merge(T1,T5)  │ ...            │
│  [95,92]          │  [92,88]          │                │
└─────────────────────────────────────────────────────────┘

Iteration 2 (stride = 2):
┌─────────────────────────────────────────────────────────┐
│  T0 merge(T0,T2)  │  T1 merge(T1,T3)  │                │
│  [95,92]          │  [92,88]          │                │
└─────────────────────────────────────────────────────────┘

Iteration 3 (stride = 1):
┌─────────────────────────────────────────────────────────┐
│  T0 merge(T0,T1)                                        │
│  [95,92] ← Final Top-2                                  │
└─────────────────────────────────────────────────────────┘

归约合并操作:
┌─────────────────────────────────────┐
│ 合并两个 Top-K 数组:                 │
│ Array A: [A0, A1, ..., A(K-1)]      │
│ Array B: [B0, B1, ..., B(K-1)]       │
│                                       │
│ Step 1: Concatenate → [A|B] (2K)    │
│ Step 2: Sort (bubble/selection)      │
│ Step 3: Take first K                 │
│ Result: [Top 0, Top 1, ..., Top K-1] │
└─────────────────────────────────────┘
```

### 执行流程

```
┌────────────────────────────────────────────────────────────────┐
│               topk_v1_reduce_kernel 执行流程                  │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 加载数据到寄存器                                            │
│     ┌─────────────────────────────────────┐                    │
│     │ if (tid < num_threads)            │                    │
│     │   load thread_tops[tid][0..K-1]   │                    │
│     │ else                              │                    │
│     │   set to -infinity                │                    │
│     └─────────────────────────────────────┘                    │
│                          │                                      │
│                          ▼                                      │
│  2. 写入共享内存                                                │
│     ┌─────────────────────────────────────┐                    │
│     │ smem[tid×K + i] = local_top[i]    │                    │
│     └─────────────────────────────────────┘                    │
│                          │                                      │
│                          ▼                                      │
│  3. __syncthreads()                                             │
│                          │                                      │
│                          ▼                                      │
│  4. 树形归约                                                    │
│     ┌─────────────────────────────────────┐                    │
│     │ for (s = blockDim.x/2; s > 0;      │                    │
│     │      s >>= 1) {                    │                    │
│     │   if (tid < s)                     │                    │
│     │     merge(smem[tid], smem[tid+s])  │                    │
│     │   __syncthreads()                  │                    │
│     │ }                                  │                    │
│     └─────────────────────────────────────┘                    │
│                          │                                      │
│                          ▼                                      │
│  5. 线程 0 输出结果                                             │
│     ┌─────────────────────────────────────┐                    │
│     │ if (tid == 0)                      │                    │
│     │   output[i] = smem[i]              │                    │
│     └─────────────────────────────────────┘                    │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## 性能分析

### 优势

| 方面 | 说明 |
|------|------|
| **时间复杂度** | $O(N \" K)$ - 单遍扫描，无全局排序 |
| **并行度** | 高 - 每个线程独立工作 |
| **实现简单** | 逻辑清晰，易于理解 |

### 瓶颈

| 问题 | 说明 | 影响 |
|------|------|------|
| **寄存器溢出** | K > 16 时，local_top 溢出到 Local Memory | 延迟飙升 |
| **全局内存写入** | 每个线程写 K 个元素，总写入量 = num_threads × K | 带宽压力大 |
| **中间数据** | thread_tops 占用大量显存 | 内存开销 |

### 数据量估算

```
假设: N = 10^7, K = 32, num_blocks = 256, num_threads = 256

总线程数 = 256 × 256 = 65,536

中间数据大小 = 65,536 × 32 × sizeof(TopKNode)
             = 65,536 × 32 × 8 bytes
             = 16,777,216 bytes
             ≈ 16 MB

全局内存写入量 = 16 MB (中间数据) + 256 bytes (输出)
```

## 关键代码片段解析

### 1. 插入排序逻辑

```cpp
// 降序维护 local_top (最大值在索引 0)
if (val > local_top[K - 1].value) {  // 比当前最小值大?
    int j = K - 1;
    // 从后往前找插入位置
    while (j > 0 && val > local_top[j - 1].value) {
        local_top[j] = local_top[j - 1];  // 后移
        j--;
    }
    local_top[j].value = val;  // 插入
    local_top[j].index = idx;
}
```

### 2. 树形归约的合并

```cpp
// 合并两个 Top-K 数组
TopKNode temp[2 * K];

// 加载
for (int i = 0; i < K; ++i) {
    temp[i] = smem[tid * K + i];
    temp[i + K] = smem[(tid + s) * K + i];
}

// 冒泡排序前 K 个
for (int i = 0; i < 2 * K - 1; ++i) {
    for (int j = 0; j < 2 * K - i - 1; ++j) {
        if (temp[j].value < temp[j + 1].value) {
            swap(temp[j], temp[j + 1]);
        }
    }
}

// 写回前 K 个
for (int i = 0; i < K; ++i) {
    smem[tid * K + i] = temp[i];
}
```

## 总结

V1 版本是 Top-K 优化的起点，核心思想：

1. **Map**: 每个线程独立处理数据，维护局部 Top-K
2. **Reduce**: 通过树形归约合并所有线程结果

**学习要点**：
- Grid-Stride Loop 设计模式
- 寄存器中维护有序数组
- Shared Memory 树形归约
- 模板编程与显式实例化

**下一版本 (V2) 改进方向**：
- 在写入全局内存前，在 Shared Memory 中进行 Block 级合并
- 减少全局内存写入量
