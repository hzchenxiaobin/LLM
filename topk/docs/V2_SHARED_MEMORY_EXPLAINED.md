# V2: 共享内存优化版 Top-K 详解

## 概述

V2 版本通过**块级共享内存合并**解决了 V1 版本的全局内存写入瓶颈问题。核心思想是：在写入全局内存之前，先在共享内存中完成块内线程的结果合并，大幅减少全局内存访问量。

---

## 核心问题：V1 的痛点

### V1 版本的写入模式

```
┌─────────────────────────────────────────────────────────────┐
│  V1: 每个线程直接写入全局内存                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Block 0:                                                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
│  │Thread 0 │  │Thread 1 │  │Thread 2 │  ... (256 threads)   │
│  │ K elems │  │ K elems │  │ K elems │                     │
│  └────┬────┘  └────┬────┘  └────┬────┘                     │
│       │            │            │                           │
│       └────────────┴────────────┘                           │
│                    │                                        │
│            ┌───────▼───────┐                               │
│            │  Global Memory │  ← 256 × K 写入/Block        │
│            │  (VRAM - 慢)   │                               │
│            └───────────────┘                               │
│                                                             │
│  全局内存写入量 = num_threads × K = 256K (假设 K=32)         │
│  中间数据量巨大，显存带宽压力大！                             │
└─────────────────────────────────────────────────────────────┘
```

### V2 的改进策略

```
┌─────────────────────────────────────────────────────────────┐
│  V2: 先在共享内存中合并，再写入全局内存                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Block 0:                                                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
│  │Thread 0 │  │Thread 1 │  │Thread 2 │  ...                 │
│  │ K elems │  │ K elems │  │ K elems │                     │
│  └────┬────┘  └────┬────┘  └────┬────┘                     │
│       │            │            │                           │
│       └────────────┴────────────┘                           │
│                    │                                        │
│            ┌───────▼───────┐                               │
│            │ Shared Memory │  ← 块内树形归约 (快 ~L1速度)    │
│            │  (~L1 Cache)   │                               │
│            └───────┬───────┘                               │
│                    │                                        │
│                    ▼ 只写 K 个元素                           │
│            ┌───────────────┐                               │
│            │  Global Memory │  ← K 写入/Block (大幅减少！)    │
│            │  (VRAM - 慢)   │                               │
│            └───────────────┘                               │
│                                                             │
│  全局内存写入量 = K (假设 K=32，减少 256 倍！)                │
└─────────────────────────────────────────────────────────────┘
```

## 代码架构详解

### 1. 设备函数：`merge_topk_in_smem`

```cpp
template <int K>
__device__ __forceinline__ void merge_topk_in_smem(TopKNode* smem, int dst_offset, int src_offset) {
    // 栈上的临时数组 (寄存器/局部内存)
    TopKNode temp[2 * K];

    // 加载两个数组
    for (int i = 0; i < K; ++i) {
        temp[i] = smem[dst_offset + i];
        temp[i + K] = smem[src_offset + i];
    }

    // 对 2K 个元素使用简单选择排序 (K <= 256, 所以 2K <= 512)
    for (int i = 0; i < K; ++i) {
        int max_idx = i;
        for (int j = i + 1; j < 2 * K; ++j) {
            if (temp[j].value > temp[max_idx].value) {
                max_idx = j;
            }
        }
        // 交换
        if (max_idx != i) {
            TopKNode swap = temp[i];
            temp[i] = temp[max_idx];
            temp[max_idx] = swap;
        }
    }

    // 将前 K 个写回目标位置
    for (int i = 0; i < K; ++i) {
        smem[dst_offset + i] = temp[i];
    }
}
```

**功能**：在共享内存中合并两个已排序的 Top-K 数组，保留前 K 个最大值。

### 2. 主内核：`topk_v2_block_kernel`

#### 阶段 1: 数据分块与 Grid-Stride 循环

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据分块示意图                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  输入数组 (N 个元素):                                            │
│  ┌────────────────────────────────────────────────────────┐    │
│  │  0    1    2    ...    N-1                              │    │
│  └────────────────────────────────────────────────────────┘    │
│                                                                 │
│  分块策略 (假设 4 个 Block, 每个 Block 负责 N/4 元素):           │
│  ┌──────────────┬──────────────┬──────────────┬──────────────┐  │
│  │   Block 0    │   Block 1    │   Block 2    │   Block 3    │  │
│  │ [0, N/4)     │ [N/4, N/2)   │ [N/2, 3N/4)  │ [3N/4, N)    │  │
│  └──────────────┴──────────────┴──────────────┴──────────────┘  │
│                                                                 │
│  每个 Block 内部使用 Grid-Stride 循环:                            │
│  ┌────────────────────────────────────────────────────────┐    │
│  │ Block 0, Thread 0 处理: idx = 0, 4, 8, 12, ...        │    │
│  │ Block 0, Thread 1 处理: idx = 1, 5, 9, 13, ...         │    │
│  │ ...                                                    │    │
│  │ Block 0, Thread T 处理: idx = T, T+4, T+8, ...          │    │
│  └────────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```cpp
// 每个线程使用 grid-stride 循环处理输入的一部分
// 但只在该块分配范围内处理
int elements_per_block = (N + num_blocks - 1) / num_blocks;
int block_start = block_id * elements_per_block;
int block_end = min(block_start + elements_per_block, N);

// 块分配范围内的 grid-stride 循环
for (int idx = block_start + tid; idx < block_end; idx += num_threads) {
    float val = input[idx];
    // 更新局部 Top-K...
}
```

#### 阶段 2: 局部 Top-K 维护（寄存器）

```
┌─────────────────────────────────────────────────────────────┐
│              每个线程维护的局部 Top-K 数组                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Thread X 的局部 Top-K (按降序排列):                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Index │  0  │  1  │  2  │  ...  │ K-2 │  K-1       │   │
│  ├───────┼─────┼─────┼─────┼───────┼─────┼────────────┤   │
│  │ Value │ max │     │     │  ...  │     │ min (-1e20)│   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  插入新元素 val 的逻辑 (降序):                              │
│                                                             │
│  1. 如果 val > local_top[K-1].value:                       │
│       - 从后向前查找插入位置                               │
│       - 边查找边移位                                       │
│       - 插入到正确位置                                     │
│                                                             │
│  示例 (K=5, 插入 val=80):                                  │
│  初始: [100, 90, 70, 50, 30]                               │
│  查找: 80 > 30 ✓, 80 > 50 ✓, 80 > 70 ✓, 80 > 90 ✗        │
│  移位: [100, 90, _, 70, 50]  ← 30 被挤出                  │
│  插入: [100, 90, 80, 70, 50]                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```cpp
// 寄存器中的局部 Top-K (按降序维护)
TopKNode local_top[K];
for (int i = 0; i < K; ++i) {
    local_top[i].value = -1e20f;  // 初始化为极小值
    local_top[i].index = -1;
}

// Grid-stride 循环处理数据
for (int idx = block_start + tid; idx < block_end; idx += num_threads) {
    float val = input[idx];

    // 如果比当前最小值大, 则插入到局部 top-K
    if (val > local_top[K - 1].value) {
        // 从末尾开始查找插入位置, 边查边移位
        int j = K - 1;
        while (j > 0 && val > local_top[j - 1].value) {
            local_top[j] = local_top[j - 1];
            j--;
        }
        local_top[j].value = val;
        local_top[j].index = idx;
    }
}
```

```
┌────────────────────────────────────────────────────────────────────────┐
│                        共享内存布局示意图                               │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  共享内存大小: blockDim.x * K * sizeof(TopKNode)                        │
│                                                                        │
│  布局 (假设 blockDim.x = 4, K = 3):                                     │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │  Thread 0    │  Thread 1    │  Thread 2    │  Thread 3           │  │
│  │  K elements  │  K elements  │  K elements  │  K elements         │  │
│  ├──────────────┼──────────────┼──────────────┼──────────────┐       │  │

│  │ smem[2]      │ smem[5]      │ smem[8]      │ smem[11]     │       │  │
│  │   (val=100)  │   (val=95)   │   (val=90)   │   (val=85)   │       │  │
│  │   (idx=0)    │   (idx=5)    │   (idx=3)    │   (idx=8)    │       │  │
│  └──────────────┴──────────────┴──────────────┴──────────────┘       │  │

```

```cpp
// 将局部 top-K 写入共享内存
for (int i = 0; i < K; ++i) {
    smem[tid * K + i] = local_top[i];
}
__syncthreads();  // 确保所有线程写入完成
```

#### 阶段 4: 树形归约（核心优化）

```
┌────────────────────────────────────────────────────────────────────────────┐
│                    树形归约过程 (Tree Reduction)                            │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  初始状态 (8 个线程, K=3 简化示例):                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │ Thread 0 │ Thread 1 │ Thread 2 │ Thread 3 │ Thread 4 │ ... Thread 7  │  │
│  │ [100,95] │ [90,85]  │ [80,75]  │ [70,65]  │ [60,55]  │ ... [20,15]   │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                            │
│  ═══════════════════════════════════════════════════════════════════════   │
│                                                                            │
│  第 1 轮 (s = 4): 线程 0-3 合并 线程 4-7 的数据                            │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  Thread 0 合并 (0,4): [100,95,90,85] → [100,95,90]                   │  │
│  │  Thread 1 合并 (1,5): [90,85,80,75] → [90,85,80]                    │  │
│  │  Thread 2 合并 (2,6): [80,75,70,65] → [80,75,70]                    │  │
│  │  Thread 3 合并 (3,7): [70,65,60,55] → [70,65,60]                    │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                            │
│  第 2 轮 (s = 2): 线程 0-1 合并 线程 2-3 的数据                            │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  Thread 0 合并 (0,2): [100,95,90,80,75,70] → [100,95,90]            │  │
│  │  Thread 1 合并 (1,3): [90,85,80,70,65,60] → [90,85,80]             │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                            │
│  第 3 轮 (s = 1): 线程 0 合并 线程 1 的数据                                │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  Thread 0 合并 (0,1): [100,95,90,90,85,80] → [100,95,90]            │  │
│  │  Thread 1 空闲                                                       │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                            │
│  最终结果: smem[0..K-1] = [100, 95, 90]                                    │
│                                                                            │
│  ⚠️ 每轮后需要 __syncthreads() 保证数据一致性                              │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

```cpp
// 使用共享内存进行块级树形归约
// 每次迭代将活动线程数减半
for (int s = num_threads / 2; s > 0; s >>= 1) {
    if (tid < s) {
        // 合并 smem[tid] 和 smem[tid + s]
        merge_topk_in_smem<K>(smem, tid * K, (tid + s) * K);
    }
    __syncthreads();  // 关键同步点！
}
```

## 性能对比

| 指标 | V1 (线程级) | V2 (共享内存) | 提升 |
|------|-------------|---------------|------|
| 全局内存写入/Block | `num_threads × K` (256K) | `K` (32) | **256× 减少** |
| 中间存储需求 | 巨大 (num_threads × K) | 极小 (num_blocks × K) | **显著降低** |
| 共享内存速度 | - | ~L1 Cache 级别 | ✅ 高速访问 |
| 同步开销 | 较少 | 需要 `__syncthreads()` | ⚠️ 略有增加 |

## 关键优化点总结

### 1. 内存层次利用

```
┌─────────────────────────────────────────────────────────────┐
│                    GPU 内存层次                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  速度层级:                                                  │
│                                                             │
│  ┌──────────────┐  最快 (~1TB/s)                            │
│  │   寄存器     │  ← 局部 Top-K 维护                        │
│  └──────────────┘                                          │
│         │                                                   │
│         ▼                                                   │
│  ┌──────────────┐  很快 (~L1 速度)                          │
│  │ 共享内存     │  ← 块级归约 (V2 核心优化)                 │
│  └──────────────┘                                          │
│         │                                                   │
│         ▼                                                   │
│  ┌──────────────┐  中等 (~L2 速度)                          │
│  │   L2 缓存    │                                           │
│  └──────────────┘                                          │
│         │                                                   │
│         ▼                                                   │
│  ┌──────────────┐  最慢 (~GB/s)                             │
│  │ 全局内存     │  ← 最终结果写入 (次数大幅减少)            │
│  │ (VRAM/GDDR)  │                                           │
│  └──────────────┘                                          │
│                                                             │
│  V2 策略: 尽量在寄存器和共享内存完成计算，                    │
│          最小化全局内存访问！                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 数据流优化

```
V1 数据流: 输入 → 寄存器计算 → 直接写全局内存 (高延迟)
           
V2 数据流: 输入 → 寄存器计算 → 共享内存归约 → 最终结果写全局内存 (优化)
                              ↑
                              └── 块内合并消除大量中间写入
```

### 3. 时间复杂度分析

| 阶段 | 复杂度 | 说明 |
|------|--------|------|
| 数据扫描 | O(N × K) | 每元素最多 K 次比较 |
| 共享内存归约 | O(num_threads × K²) | 每次合并 O(K²) 选择排序 |
| 最终归约 | O(num_blocks × K²) | 同上 |
| **总复杂度** | **O(N × K + (T+B) × K²)** | 当 N >> K 时接近 O(N × K) |

## 关键代码片段解析

### `__syncthreads()` 的重要性

```cpp
// ⚠️ 错误示例：缺少同步会导致数据竞争！
for (int s = num_threads / 2; s > 0; s >>= 1) {
    if (tid < s) {
        merge_topk_in_smem<K>(smem, tid * K, (tid + s) * K);  // 写入 smem
    }
    // ❌ 缺少 __syncthreads()！
    // 下一轮迭代时，其他线程可能还在读取上一轮的数据！
}

// ✅ 正确示例：每次归约后必须同步
for (int s = num_threads / 2; s > 0; s >>= 1) {
    if (tid < s) {
        merge_topk_in_smem<K>(smem, tid * K, (tid + s) * K);
    }
    __syncthreads();  // 确保所有线程完成写入后再继续
}
```

### 选择排序的合理性

```cpp
// 为什么用选择排序而不是更高效的算法？
// 1. K 很小 (<= 256, 2K <= 512)
// 2. 数据在栈上（寄存器/局部内存），访问很快
// 3. 选择排序实现简单，常数小
// 4. 只需要前 K 个有序，不需要全排序

// 优化点：只进行 K 轮选择，而不是 2K 轮
for (int i = 0; i < K; ++i) {  // 只取前 K 个
    int max_idx = i;
    for (int j = i + 1; j < 2 * K; ++j) {
        if (temp[j].value > temp[max_idx].value) {
            max_idx = j;
        }
    }
    // ...
}
```

## 总结

V2 版本的核心改进是**利用共享内存进行块级数据合并**，将全局内存写入量从 `num_threads × K` 减少到 `num_blocks × K`。在典型的 256 线程/Block 配置下，这是 **256 倍的写入量减少**。

### 学习要点

1. **内存层次意识**：理解寄存器 → 共享内存 → L2 → 全局内存的速度差异
2. **归约模式**：掌握树形归约（Tree Reduction）这一经典的并行算法模式
3. **同步机制**：正确使用 `__syncthreads()` 避免数据竞争
4. **分治策略**：Map-Reduce 思想在 GPU 并行计算中的应用

### 适用场景

- ✅ 中小规模 K 值 (K <= 256)
- ✅ 需要减少全局内存带宽压力的场景
- ✅ 适合学习 GPU 内存层次和同步机制

### 下一步

V3 版本将进一步优化，使用 **Warp Shuffle 原语** 在寄存器级别完成 Warp 内的数据交换，彻底消除共享内存的同步开销！
