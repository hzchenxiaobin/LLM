3. Pipeline 优化内核 (V5.3)

针对重复数据访问场景，使用共享内存缓存数据块：

📥
Phase 1: 协作加载
所有线程协作将数据块从全局内存加载到共享内存。 顺序访问模式，100% 缓存行利用率。
⬇
⏳
同步点
__syncthreads() 确保所有线程完成加载。 数据现在在共享内存中可用。
⬇
⚙️
Phase 2: 处理
所有线程从共享内存读取数据（L1 速度）。 处理完成后再次同步，进入下一块。 
// V5.3: Pipeline 优化内核
template <int K, int BLOCK_THREADS, int CHUNK_SIZE>
__global__ void topk_v5_pipeline_kernel(
    const float* __restrict__ input,  // 优化: 无指针别名
    int N, TopKNode* block_tops) 
{
    // 额外共享内存用于数据缓存
    float* s_data = (float*)&smem[WARPS_PER_BLOCK * K];

    // 分块处理
    for (int base_idx = block_start; base_idx < block_end; base_idx += CHUNK_SIZE) {
        
        // 1. 协作加载到共享内存
        for (int i = tid; i < chunk_size; i += BLOCK_THREADS) {
            s_data[i] = input[base_idx + i];  // 顺序读取，缓存友好
        }
        __syncthreads();

        // 2. 从共享内存处理 (L1 速度!)
        for (int i = tid; i < chunk_size; i += BLOCK_THREADS) {
            float val = s_data[i];  // ~2 周期 vs ~100 周期
            insert_topk_blackwell<K>(local_top, val, global_idx);
        }
        __syncthreads();
    }
}
Pipeline 优势:
• 顺序读取：缓存行利用率 100%
• 共享内存速度：~2 周期 vs 全局内存 ~100 周期
• Blackwell 96MB L2：可缓存更多 chunk
• 适合场景：重复 Top-K 查询、批量处理

5.3 优化的归并算法

// V5: 双路归并 O(K) 替代选择排序 O(K²)
TopKNode merged[2 * K];
int a = 0, b = 0;

for (int i = 0; i < 2 * K; i++) {
    // 比较两个有序数组的当前元素
    if (b >= K || local_top[a].value > received[b].value) {
        merged[i] = local_top[a++];  // 取较大者
    } else {
        merged[i] = received[b++];
    }
}

// 复杂度对比:
// • V3 选择排序: O(K²) = 256 次比较 (K=16)
// • V5 双路归并: O(2K) = 32 次比较 (K=16)
// • 加速: 8×，且分支更可预测

6. 总结

V5 是针对 Blackwell 架构的深度微优化，核心改进：

🔄
#pragma unroll
强制循环展开，消除循环开销，最大化指令级并行 (ILP)。
🎯
__restrict__
消除指针别名，编译器可激进优化内存访问，提高缓存命中率。
⚡
双路归并
O(K) 算法替代 O(K²) 选择排序，8× 加速，分支更可预测。
📦
Pipeline 内核
共享内存缓存数据块，适合重复访问场景，接近 L1 速度。
