## 1. CUDA 线程层级结构

### 1.1 三层架构

```
┌─────────────────────────────────────────────────────────┐
│                    Grid (整个 GPU)                       │
│   ┌─────────────────┐  ┌─────────────────┐              │
│   │   Block (0,0)   │  │   Block (1,0)   │   ...        │
│   │  ┌───────────┐  │  │  ┌───────────┐  │              │
│   │  │ Warp 0    │  │  │  │ Warp 0    │  │              │
│   │  │ (32线程)  │  │  │  │ (32线程)  │  │              │
│   │  ├───────────┤  │  │  ├───────────┤  │              │
│   │  │ Warp 1    │  │  │  │ Warp 1    │  │              │
│   │  └───────────┘  │  │  └───────────┘  │              │
│   └─────────────────┘  └─────────────────┘              │
└─────────────────────────────────────────────────────────┘
```

### 1.2 各层级对比

| 层级 | 大小 | 同步方式 | 内存访问 | 硬件映射 |
|------|------|----------|----------|----------|
| **Warp** | 32 线程 | 隐式同步 | 寄存器 | SM 内执行单元 |
| **Block** | 最多 1024 线程 | `__syncthreads()` | 共享内存 | 分配到单个 SM |
| **Grid** | 无限（受硬件限制） | 无法直接同步 | 全局内存 | 跨多个 SM |

### 1.3 线程索引计算

```cuda
__global__ void kernel() {
    // Grid 级别
    int blockId = blockIdx.x + blockIdx.y * gridDim.x;
    
    // Block 级别
    int threadId = threadIdx.x + threadIdx.y * blockDim.x;
    
    // Warp 级别
    int warpId = threadId / 32;
    int laneId = threadIdx.x % 32;
    
    // 全局线程 ID
    int globalId = blockId * (blockDim.x * blockDim.y) + threadId;
}
```

## 2. SM 架构与资源限制

### 2.1 RTX 5090 SM 规格

| 资源 | RTX 5090 配置 | 作用 |
|------|--------------|------|
| **SM 数量** | 170 | 并行计算单元 |
| **每 SM CUDA Core** | 128 个 | FP32 运算 |
| **每 SM Tensor Core** | 4 个 | 矩阵加速 |
| **寄存器文件** | 256 KB/SM | ~65,536 个 32-bit 寄存器 |
| **共享内存** | 164 KB/SM | Block 内数据共享 |
| **每 SM 最大线程** | 2048 | 并发执行上限 |
| **每 SM 最大 Warp** | 64 | 调度单位 |

### 2.2 Occupancy（占用率）

**定义**：实际运行的 Warp 数 / 每 SM 最大 Warp 数

**计算公式**：
```
Occupancy = (实际运行 Warp 数) / 64 × 100%

限制因素（取最小值）：
- 寄存器限制：256KB / (每线程寄存器 × 32 × 4B)
- 共享内存限制：164KB / 每 Block 共享内存
- 线程限制：2048 / Block 大小
```

**常用配置参考**：

| 每线程寄存器 | Occupancy | 适用场景 |
|-------------|-----------|---------|
| 64 | 50% | 平衡型 |
| 90 | 33% | GEMM 优化（推荐） |
| 128 | 25% | 高计算强度 |

## 3. 内存层次结构

### 3.1 三级内存对比

| 层级 | 延迟 | 带宽 | 容量 | 用途 |
|------|------|------|------|------|
| **寄存器** | ~1 cycle | ~10+ TB/s | ~256 B/线程 | 累加器、临时变量 |
| **共享内存** | ~20-30 cycles | ~10+ TB/s | ~164 KB/SM | Block 数据缓存 |
| **全局内存** | ~300-500 cycles | ~1.8 TB/s | ~32 GB | 矩阵存储 |

### 3.2 Shared Memory Bank 结构

```
共享内存组织：
- Bank 数量：32（与 Warp 大小匹配）
- 每个 Bank 宽度：4 bytes
- 地址映射：Bank ID = (地址 / 4) % 32

Bank Conflict：当 Warp 中多个线程同时访问同一 Bank 的不同地址时，
访问会被串行化，导致性能下降。
```

## 4. GEMM 基础概念

### 4.1 标准 GEMM 公式

$$C = \alpha \cdot A \times B + \beta \cdot C$$

- $A$: $M \times K$ 矩阵
- $B$: $K \times N$ 矩阵
- $C$: $M \times N$ 矩阵
- 计算复杂度：$O(M \times N \times K)$

### 4.2 Arithmetic Intensity（计算强度）

**定义**：AI = 浮点运算次数 / 内存访问字节数

**计算**：
```
对于 M=N=K 的 GEMM：
- 浮点运算：2 × M³
- 内存访问（优化后）：3 × M² × 4 bytes
- AI ≈ M / 6 FLOPs/byte

当 M=4096 时：AI ≈ 683 FLOPs/byte
```

### 4.3 Roofline 模型

```
Performance = min(Peak FLOPS, Memory BW × AI)

RTX 5090 参数：
- Peak FP32: 104.9 TFLOPS
- Memory BW: 1,792 GB/s
- Ridge Point: 104.9 / 1.792 ≈ 58.5 FLOPs/byte

当 AI > 58.5 时：计算受限（GEMM 属于此类）
当 AI < 58.5 时：内存受限
```

## 关键要点总结

| 概念 | 核心要点 |
|-----|---------|
| **Warp** | 32 线程同时执行，最小调度单位 |
| **Block** | 最多 1024 线程，可共享内存和同步 |
| **Occupancy** | 25-50% 对 GEMM 通常是最佳选择 |
| **AI** | 通过分块优化从 <1 提升到 >100 |
| **Bank Conflict** | 用 Padding 避免，提升共享内存效率 |
